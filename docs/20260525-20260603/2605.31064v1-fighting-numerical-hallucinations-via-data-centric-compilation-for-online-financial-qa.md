## 摘要
大型语言模型（LLMs）显著推动了在线数据服务的发展，特别是在金融问答（FinQA）领域。然而，这类系统仍容易出现数值推理幻觉，这在风险较高的金融应用中严重损害了可靠性。尽管检索增强生成（RAG）已被广泛采用以将回答建立在外部知识之上，但它引入了三个持续存在的挑战：噪声敏感性、计算脆弱性以及可审计性危机。现有的以模型为中心的方法主要侧重于孤立地优化检索器或生成器，仍难以以集成的方式解决这些问题。在本工作中，我们开创性地提出了一种以数据为中心的范式，并设计了一个新颖的框架——数据中心化推理编译器（DCRC）。该框架通过三个连贯的阶段运作：（1）对抗性数据构建，合成带有可控噪声的训练样本以培养鲁棒性；（2）多阶段训练，训练出一个数据中心化结构化智能体（DSA），能够进行显式证据审计和程序合成；（3）编译-执行推理过程，其中DSA将用户查询和检索到的文档转换为可验证、可执行的推理程序。这一数据驱动的框架从设计上确保忠实的数值推理。我们在既有的离线基准上进行了广泛实验，并通过部署在实际的在线金融问答系统中进一步验证了我们的框架。

## Abstract
Large Language Models (LLMs) have significantly advanced online data services, particularly in the domain of financial question answering (FinQA). However, such systems remain susceptible to numerical reasoning hallucinations, which critically undermine reliability in high-stakes financial applications. Although retrieval-augmented generation (RAG) has been widely adopted to ground responses in external knowledge, it introduces three persistent challenges: noise sensitivity, calculation fragility, and an auditability crisis. Existing model-centric approaches, which primarily focus on optimizing either the retriever or generator in isolation, still struggle to address these issues in an integrated manner. In this work, we pioneer a data-centric paradigm and propose a novel framework, the Data-centric Reasoning Compiler (DCRC). The framework operates through three cohesive phases: (1) adversarial data construction, which synthesizes training examples with controlled noise to teach robustness; (2) multi-stage training that cultivates a Data-centric Structuring Agent (DSA) capable of explicit evidence auditing and program synthesis; and (3) a compile-and-execute inference process, where the DSA transforms user queries and retrieved documents into verifiable, executable reasoning programs. This data-driven framework ensures faithful numerical reasoning by design. We conduct extensive experiments on established offline benchmarks and further validate our framework through deployment in a real-world online financial QA system.

## 2. 论文提出的方法论：核心思想与关键技术细节  
**核心思想**：提出**数据中心化推理编译器（DCRC）** 框架，将复杂推理任务编译为“可审计、结构化、确定性执行”的中间程序，从源头杜绝推理幻觉。  

**关键技术流程**（分三个阶段）：  
- **阶段1：对抗性数据构建（教会模型“学会拒绝”）**  
  - **异构共识挖掘**：利用多个异构 LLM（如 GPT-5.2、Gemini-3 Pro、Claude-4.5 Opus）组成教师委员会，对每个检索证据进行投票，仅将多数模型一致接受的证据标记为“正例（ACCEPT）”。  
  - **硬负样本注入**：基于正例，通过受控扰动生成三种高难度负样本：  
    - *概念转移*：使用定义不同但语义相似的数值（如“经营利润”代替“归母净利润”），标记为 `[Concept_Shift]`。  
    - *时间错配*：使用不同年份的同类型数值，标记为 `[Time_Mismatch]`。  
    - *实体混淆*：使用不同公司或实体的数值，标记为 `[Entity_Mismatch]`。  
  - **一致性过滤**：利用验证模型检查每个样本的“决策标签”与“拒绝理由”之间是否存在逻辑矛盾，确保数据质量。  

- **阶段2：多阶段模型训练（培养数据中心化结构化代理 DSA）**  
  - **结构感知监督微调**：在标准 SFT 基础上，对关键令牌（如 `ACCEPT`、`REJECT`）施加 **2.0 倍损失权重**，强制模型关注证据审计决策的边界学习。  
  - **群组相对策略优化（GRPO）+ 多维奖励**：设计五维奖励函数：  
    - *格式奖励*：输出是否为合法 JSON。  
    - *检索奖励*：使用 F1 衡量证据审计的精准度和召回率。  
    - *一致性惩罚*：若模型在“拒绝”的证据上又将其数值用于计算，则给予高额负奖励（如 -5.0）。  
    - *可执行奖励*：生成的符号化计算逻辑能否被执行器正确执行。  
    - *准确度奖励*：最终数值结果是否与真值一致。  

- **阶段3：编译-执行推理（确定性化推理过程）**  
  - **DSA 编译**：在推理时，DSA 将查询和检索到的文档转换为结构化中间表示，包含：  
    - *子任务分解*：将复杂问题拆解为原子目标。  
    - *共享证据池*：每条证据附带面向不同子任务的“接受 / 拒绝”决策及结构化原因。  
    - *符号化计算逻辑*：使用注册库中的算子（如 `subtract`、`divide`）编写计算式。  
  - **确定性符号执行**：  
    - 遍历证据池，根据决策标签将证据绑定到对应子任务。  
    - 执行一致性检测（如冲突或缺失），然后在一个安全的沙箱中调用预注册算子完成计算，生成**完整的、可回溯的审计日志**。

## 7. 优点：方法与实验设计的亮点  
- **范式创新**：率先将金融数值推理从“模型中心”转向“数据中心”，并引入“编译‑执行”引擎，实现推理与计算的彻底分离，使计算变成确定性过程。  
- **系统性应对三个核心挑战**：通过“硬负样本注入”克服噪声敏感性，通过“符号算子执行”消除计算脆弱性，通过“结构化审计日志”解决可审计性危机，形成闭环。  
- **透明且可解释的审计机制**：模型不仅给出答案，还显式记录每条证据的“接受 / 拒绝”理由，为高风险金融决策提供完整追溯链。  
- **实验立体且全面**：覆盖离线基准、消融分析、范式对比与真实线上部署，多维度验证了方法的有效性与实用性。

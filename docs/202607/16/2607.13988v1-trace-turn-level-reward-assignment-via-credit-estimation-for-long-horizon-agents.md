## 摘要
多回合智能体通过工具交互的长序列解决复杂任务并最终给出答案，这使得信用分配成为后训练中的一个根本挑战。结果奖励为短程推理提供了可靠监督，但随着轨迹增长到几十或几百次工具调用，奖励变得稀疏且高方差。它们还可能有误导性：一个失败的 rollout 可能包含许多有用的动作，使智能体更接近目标，但仅基于结果的训练却将它们与最终错误赋予相同的负优势。我们提出 TRACE（基于信用估计的回合级奖励分配），一种用于智能体强化学习的密集信用分配方法。TRACE 将 rollout 表示为工具调用边界上的状态转移，从冻结的参考模型中获取正确答案的对数概率，将其转换为对数比率状态值，并以这些值的时序差分变化作为每个动作的奖励。这不需要额外的评论家或过程标签训练，并且其单步对数比率时序差分组件可在冗余的工具调用间 telescoping（抵消）。在长程复杂搜索中，TRACE 仅使用纯强化学习显著提高了基础模型的工具使用能力，无需冷启动监督微调阶段、智能体中训练阶段或基于实时网络数据的训练。在闭网 BrowseComp-Plus 基准上，它将 Qwen3-4B 的得分从 7.2 提高到 35.6，将 Qwen3-30B-A3B 从 8.4 提高到 42.6。学习到的搜索行为也迁移到开网基准，并且学习曲线显示在强化学习训练期间改进更早、收敛更快。

## Abstract
Multi-turn agents solve complex tasks through extended sequences of tool interactions before producing a final answer, making credit assignment a fundamental challenge during post-training. Outcome rewards provide reliable supervision for short-horizon reasoning, but become sparse and high-variance as trajectories grow to tens or hundreds of tool calls. They can also be misleading: a failed rollout may contain many useful actions that move the agent closer to the goal, yet outcome-only training assigns them the same negative advantage as the eventual mistake. We propose TRACE (Turn-level Reward Assignment via Credit Estimation), a dense credit-assignment method for agentic reinforcement learning. TRACE represents rollouts as state transitions at tool-call boundaries, obtains gold-answer log-probabilities from a frozen reference model, transforms them into log-ratio state values, and derives per-action rewards as Temporal-Difference changes in those values. This requires no additional critic or process-label training, and its one-step log-ratio TD component telescopes across redundant tool calls. On long-horizon complex search, TRACE substantially improves base-model tool-use ability using pure RL, without a cold-start supervised fine-tuning stage, an agentic mid-training stage, or training on live-web data. On the closed-web BrowseComp-Plus benchmark, it raises Qwen3-4B from $7.2$ to $35.6$ and Qwen3-30B-A3B from $8.4$ to $42.6$. The learned search behavior also transfers to open-web benchmarks, and the learning curves show earlier improvement and faster convergence during RL training.

## 2. 论文提出的方法论
### 核心思想
- 将每条多轮轨迹分解为**工具调用边界上的状态转移**，利用一个**冻结的参考模型**（策略初始化的副本）计算在每个状态前缀下产生正确答案的对数概率，以此评估“离正确答案还有多远”，并通过**时序差分（TD）变化**将答案可预测性的增益转化为每个工具调用的**回合级奖励**。

### 关键技术细节（以文字说明）
- **状态定义**：给定轨迹前缀 \(S_k\)（包含问题及前 \(k\) 个动作-观察对），TRACE 构造每个工具调用后的状态。
- **状态价值计算**：
  1. 用冻结参考模型计算 \(S_k\) 下正确答案 \(y^*\) 的平均对数概率 \(\bar{\ell}_k\)。
  2. 定义剩余差距 \(d_k = -\bar{\ell}_k + \epsilon\)，然后计算**对数比率状态价值**：
     \[
     V(S_k) = \log \frac{d_0}{d_k} = \log \frac{-\bar{\ell}_0 + \epsilon}{-\bar{\ell}_k + \epsilon}
     \]
     初始状态 \(V(S_0) = 0\)，价值越大表示已越接近答案。
- **回合级 TD 奖励**：
  - **单步信用**：\(\delta_k = V(S_{k+1}) - V(S_k) = \log \frac{d_k}{d_{k+1}}\)，正值表示该工具调用使答案更可预测，该组件具有**可 telescoping 性质**（累加只与首尾状态有关，防止冗余动作的奖励膨胀）。
  - **K 步截断 TD**：为处理延迟效应（如搜索后需打开页面才有证据），对当前回合引入未来 \(K\) 步的折扣 TD 变化作为局部进度信用 \(c^{(K)}_{g,k}\)。
  - **终端锚定**：当 K 步窗口触及轨迹末尾时，额外加入**结果优势**的折扣项，使回合奖励与最终正确性对齐。
- **与结果奖励混合**：最终的每 token 优势为：
  \[
  \hat{A}_{g,t} = \alpha_{\text{out}} A^{\text{out}}_g + \alpha_{\text{turn}} r^{\text{turn}}_{g, \text{turn}(t)}
  \]
  其中 \(A^{\text{out}}_g\) 是标准 GRPO 的组内相对优势。利用裁剪后的 GRPO 目标进行策略优化。

### 算法流程（文字概括）
1. 对每个提示，采样多条轨迹。
2. 计算结果奖励 \(R_g\) 和结果优势 \(A^{\text{out}}_g\)。
3. 对每条轨迹，构造答案就绪的前缀状态 \(S_{g,k}\)。
4. 用冻结参考模型计算每个状态下的 \(\bar{\ell}_{g,k}\)，得到 \(V_{g,k}\)。
5. 计算相邻状态的单步 TD 误差 \(\delta_{g,k}\)。
6. 根据 K 步窗口和终端锚定得到回合级奖励 \(r^{\text{turn}}_{g,k}\)。
7. 将回合奖励与结果优势混合，得到每个助手 token 的优势 \(\hat{A}_{g,t}\)。
8. 使用裁剪 PPO 目标更新策略。

## 6. 论文的主要结论与发现
- TRACE 在**仅使用纯 RL**（无冷启动 SFT、中训练、在线数据）的条件下，**大幅提升**了基础模型的工具使用能力：在 BrowseComp‑Plus 上将 Qwen3-4B 从 7.2 提高到 35.6，将 Qwen3-30B-A3B 从 8.4 提高到 42.6。
- 该方法在含外部 API 的开网基准和跨语言任务上**表现出迁移性**，学习到的搜索策略并非仅拟合离线语料。
- 与 GRPO 等结果奖励方法相比，**学习曲线改善更早、收敛更快**，回合级密集信用信号有助于早期训练阶段探索和扩展交互长度。
- 消融实验表明，**对数比率形式的信用分配最有效**，适度的 K 步传播优于无传播或过度传播，冻结的初始参考模型足以提供可靠的状态价值估计。

## 7. 优点
- **无需额外昂贵组件**：不训练评论家、过程奖励模型，也不需要步骤级标签或强法官，实现简单，仅依赖一个冻结的参考模型。
- **信用分配设计精妙**：使用对数比率价值并利用 TD 变化，既保留了时序上的可相加性（防止冗余奖励），又能处理工具调用间的延迟影响。
- **保留结果锚点**：将回合级密集奖励与结果优势结合，既提供了中间反馈，又不会偏离最终答案正确性的训练目标。
- **实验控制严格**：受控基线在完全相同的环境、数据和奖励下比较，可靠地证明了密集信用信号的价值。
- **分析深入**：从学习曲线、工具调用次数趋势到详细的定性案例，多角度揭示了 TRACE 如何影响训练行为和模型决策。

## 8. 不足与局限
- **任务范围受限**：当前设计只适用于**答案短小、可自动验证**的任务，对于需要生成长答案、结构化补丁或多文件输出的编程或开放域对话场景，正确答案的对数概率可能不是可靠的状态价值指标。

## 摘要
基于LLM的数据智能体在数据驱动决策中扮演着越来越重要的角色。然而，现有数据智能体难以泛化到未见过的数据环境和分析工作流，特别是在异构企业环境中。这催生了对合成高质量数据智能体轨迹的日益增长的需求，这些轨迹能够捕捉给定数据环境下的复杂分析工作流。此类轨迹支持两个关键的下游用途：它们可以作为有监督微调（SFT）数据，使数据智能体模型适应目标领域；也可作为上下文学习（ICL）示例，指导通用LLM在陌生数据环境中进行推理。因此，我们引入了TOFFEE，一个通过蒙特卡洛树搜索（MCTS）结合自适应模型选择和跨任务前缀重用来从给定数据环境合成高质量数据智能体轨迹的系统。我们展示了TOFFEE能够有效地为异构环境中的复杂分析任务生成可扩展的轨迹数据。在本演示中，我们介绍了TOFFEE的系统框架，包括其任务池构建、轨迹探索器和学习的成本模型。我们还介绍了TOFFEE的Web界面及其工作流，并展示了两个端到端场景：用于数据智能体微调的轨迹合成，以及基于演示增强的数据智能体推理。

## Abstract
LLM-powered data agents are playing an increasingly important role in data-driven decision making. However, existing data agents struggle to generalize to unseen data environments and analytical workflows, especially in heterogeneous enterprise settings. This creates a growing need for synthesizing high-quality data agent trajectories that capture complex analytical workflows for given data environments. Such trajectories support two key downstream uses: they can serve as supervised finetuning (SFT) data that adapts data agent models to the target domain, and as in-context learning (ICL) demonstrations to guide general-purpose LLMs in unfamiliar data environments. Thus, we introduce TOFFEE, a system for synthesizing high-quality data agent trajectories from given data environments via Monte Carlo Tree Search (MCTS) with adaptive model selection and cross-task prefix reuse. We show that TOFFEE can effectively generate scalable trajectory data for complex analytical tasks across heterogeneous environments. In this demonstration, we present the system framework of TOFFEE, including its task pool construction, trajectory explorer, and learned cost model. We also introduce the web interface of TOFFEE and its workflow, and demonstrate two end-to-end scenarios: trajectory synthesis for data agent finetuning, and demonstration-augmented data agent reasoning.

### 2. 论文提出的方法论
TOFFEE 的架构分为三个核心阶段：

- **任务池构建（Task Pool Construction）**
  - **目标**：从用户提供的异构数据环境（数据库、CSV、JSON 等）中自动生成**真实、可解且有意义**的分析问题，避免 LLM 直接基于 Schema 生成问题时的浅层或无法作答的缺陷。
  - **流程**：
    1. **环境剖析**：收集 Schema 元数据和样本行。
    2. **依赖发现**：运行预定义的 SQL 探针，发现表间依赖（如共享键、数据异常模式）并将其构建成依赖图。
    3. **问题生成**：将每条依赖转化为具体的数据分析问题，由 LLM 用自然语言表述。
    4. **执行级准入验证**：对每个候选问题，按智能体的方式回放解决步骤，确认其具有**确定性的答案**，并排除那些无需访问数据就能回答的问题，防止污染下游评测。

- **轨迹探索器（Trajectory Explorer）**
  - **核心思想**：将轨迹合成看作树搜索问题：节点表示分析状态，边表示一次工具调用（如执行 SQL 查询、运行 Python 脚本），该调用由特定的 LLM 配置生成。
  - **关键技术**：
    - **MCTS 搜索**：不断扩展多个分支，保留最有潜力的路径，避免了单次生成“一步失败全盘丢弃”的缺陷。
    - **错误恢复分支**：当某步出现执行错误时，Explorer 不会中止，而是开辟修复路径，生成的“错误—诊断—修复”序列将作为训练数据，让下游智能体学会从真实错误中恢复。
    - **跨任务前缀重用**：同一数据环境下不同任务共享 Schema 发现、数据剖析等公共操作，避免重复计算。

- **学习的成本模型（Learned Cost Model, LCM）**
  - **作用**：在轨迹搜索过程中，根据当前分析状态和剩余预算，为每一步动态选择最佳的 LLM 模型、推理强度、上下文长度和分支宽度。
  - **机制**：
    - 提取状态特征（轨迹进度、Schema 覆盖率、近期错误历史等）。
    - 对每种可行的（操作、模型、配置）组合，计算**预测奖励 μ** 与**探索奖励 σ**，兼顾开采与探索。
    - 通过岭回归在线更新 LCM 参数，随着合成进行，越来越能准确地将例行步骤（如读取表名）路由到低成本小模型，将复杂推理（如多表连接）路由到高成本大模型。

- **轨迹导出**：合成的轨迹以多轮对话的形式导出，可直接作为 SFT 训练样本，或作为 ICL 演示被检索系统注入到新任务的提示中。

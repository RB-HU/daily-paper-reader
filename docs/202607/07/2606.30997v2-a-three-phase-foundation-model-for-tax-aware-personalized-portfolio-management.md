## 摘要
我们提出了一种用于个性化投资组合管理的三阶段深度强化学习系统，解决了所有先前金融强化学习工作中共有的三个局限性：1) 股票代码锁定，2) 单一化目标，以及 3) 静态用户模型。第一阶段通过在多资产语料库上的自监督学习预训练一个与股票代码无关的跨资产编码器，并利用 Chronos（一种基于 T5 的时间序列基础模型）的一个冻结并行分支进行增强，通过可学习的门控机制融合。据我们所知，这是时间序列基础模型在投资组合管理强化学习中的首次应用。该编码器通过一个 50 维的可观测元数据向量泛化到任何可公开交易的资产，并且对于新的股票代码无需重新训练。第二阶段使用 PPO 对 MoE（专家混合）投资组合行动者-评论家进行微调，奖励以目标为条件，同时服务于每回合采样的六个不同投资目标：短期 alpha、短期收益、长期收益、资本保值、税损收割和仅长期收益。MoE 架构将每个目标分配给一个专门的专家头（动量、增长、防御、税收感知），并有一个可学习的意图路由器根据活跃目标和当前市场状态混合专家，这消除了跨目标的梯度冲突。第三阶段添加了一个轻量级的个性化层，在推理时通过一个 76 参数的 LoRA 模块进一步适应每个个体，该模块基于真实的经纪交易历史进行微调，从展现的交易行为而非问卷中推断投资目标。自然语言意图解析器将自由形式的目标直接转换为结构化的投资目标参数。

## Abstract
We present a three-phase deep reinforcement learning system for personalized portfolio management that addresses three limitations shared by all prior financial RL work: 1) ticker lock-in, 2) monolithic objectives , and 3) static user models. Phase 1 pretrains a ticker-identity-free cross asset encoder via self-supervised learning on a multi-asset corpus, augmented by a frozen parallel branch using Chronos, a T5-based time series foundation model, fused via a learned gating mechanism. To our knowledge, this is the first application of a time series foundation model to portfolio management RL. The encoder generalizes to any publicly traded asset via a 50-dimensional observable metadata vector that requires no retraining for new tickers. Phase 2 fine-tunes a MoE (Mixture of Experts) portfolio actor critic with PPO under an objective-conditioned reward that simultaneously serves six distinct investment goals sampled per episode: short-term alpha, short-term gain, long-term gain, capital preservation, tax-loss harvesting, and long-term-gains-only. A MoE architecture assigns each objective to a specialized expert head (momentum, growth, defensive, tax-aware), and a learned intent router blends experts based on the active objective and current market regime, which eliminates cross-objective gradient conflict. Phase 3 adds a lightweight personalization layer further adapted at inference time to each individual via a 76-parameter LoRA module fine-tuned on real brokerage transaction history, inferring investment objectives from revealed trading behavior rather than questionnaires. A natural language intent parser converts free-form goals directly into structured investment objective parameters.

本文聚焦于**个性化投资组合管理**中三个被以往方法普遍忽视的根本性难题：
- **资产代号锁定（Ticker Lock-in）** ：传统金融强化学习（RL）模型训练时依赖固定的资产集合，更换或新增股票必须重新训练，无法泛化。
- **单一化目标（Monolithic Objectives）** ：几乎所有前期工作仅优化单一指标（如夏普比率），无法同时满足投资者差异化的目标（如短期alpha、长期增长、税务优化等）。
- **静态用户模型（Static User Models）** ：用户偏好通过一次性问卷获取，之后不再更新，无法反映其真实交易行为中的动态意图。

**整体含义**：论文旨在构建一个能摆脱资产种类限制、动态理解用户意图，并同时处理多种投资目标的统一框架，从而填补专业量化工具与简单零售工具之间的空白。

### 2. 方法论

整体框架采用三阶段设计，依次为资产无关编码器预训练、多目标MoE投资策略微调、以及基于交易行为的个性化适配。

- **阶段一：自监督跨资产编码器预训练（Phase 1 SSL Pretraining）**
  - **核心思想**：消除对固定股票代号的依赖，代之以一个**50维的可观测元数据向量**（包含行业、基本面、分析师共识、期权信号、财报日历、内部人情绪等），让模型通过资产特征而非名称来理解资产。
  - **编码器融合**：主线为自监督市场编码器（Transformer），并**融合冻结的 Chronos‑T5‑Small 时间序列基础模型**（在超过1000亿数据点上预训练）作为一个并行的特征分支。两者通过一个可学习的门控机制组合，元数据向量随后以加法注入。这是时序基础模型在投资组合RL中的首次应用。
  - **预训练目标**：包含三个标准自监督损失（下一时点收益预测、掩码特征重建、市场状态分类）以及一个**关键的跨资产对比损失**。该对比损失强制不同股票的表征相互分离，解决了仅用标准目标时所有股票表征趋同的“表征坍塌”问题（余弦相似度从0.96降至0.24）。
  - **关键设计**：编码器仅依赖元数据，推理时可接受任意数量的股票，无需重训。

- **阶段二：目标条件化组合强化学习与专家混合（Phase 2 Objective‑Conditioned MoE‑RL）**
  - **核心思想**：在一条策略中同时服务六种投资目标（短期alpha、30天收益、1年收益、资本保值、税损收割、仅长期收益），且避免目标间梯度冲突。
  - **MoE 策略架构**：设置四个专家头（动量、增长、防御、税务感知），每个专家专门负责部分目标（见表2）。一个可学习的**意图路由器**根据当前激活的目标（one-hot）和市场状态（编码器输出）生成专家混合权重，输出最终的投资组合配置。
  - **奖励设计**：奖励函数以目标为条件（每回合随机采样一个目标），包含基于目标的夏普项、组合集中度惩罚、换手惩罚、现金拖累惩罚、再部署奖励等，并针对税务目标设计了特殊的收益项（如税损收割时机奖励）。
  - **训练策略**：采用**六阶段课程学习**解决直接多目标训练的不稳定问题。先依次单独训练动量、增长、防御专家（其他专家冻结），然后加入监督的路由损失联合训练所有专家和路由器，最后通过**专家嫁接（Expert Grafting）** 将课程早期最佳专家权重复制到最终路由器上，恢复各专家性能。
  - **显式现金分配**：在组合权重softmax中加入可学习的现金令牌，迫使模型主动决定现金持仓，而不是被动持有现金。

- **阶段三：基于交易行为的个性化适配（Phase 3 Personalization with LoRA）**
  - **核心思想**：从用户真实经纪账户的交易历史中提取行为轮廓（如持有时长、卖出亏损比例等），推断其真实投资偏好，从而个性化调整模型，无需重新训练全部参数。
  - **实现**：在意图路由器上添加一个极轻量的**76 参数 LoRA 模块**，仅根据16维用户行为向量对路由权重进行微小偏移。共享编码器和所有专家头完全冻结。
  - **自然语言输入**：通过意图解析器将“3年后买房”等自由文本转换为结构化的投资目标和参数。
  - **信任优先 UX**：先展示推断的用户档案，待用户确认后才实施个性化适配。

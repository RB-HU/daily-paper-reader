## 摘要
训练调用API的大型语言模型（LLM）智能体需要大量高质量的交互轨迹。然而，大规模收集此类数据通常需要具备可执行API和真实预填充后端数据库的完整实现环境，这成为可扩展性的主要瓶颈。为克服这一挑战，我们提出了一种无环境合成数据生成方法，将LLM用作即时的数字世界模型。仅凭API规范，我们的方法就能生成模拟智能体与有状态环境交互的轨迹。具体而言，LLM首先生成可利用给定API求解的多样化任务。随后，教师智能体迭代求解每个任务，同时LLM模拟器根据任务上下文和模拟历史生成连贯的合成API响应。最后，LLM裁判对轨迹进行过滤，以确保数据集的质量。我们在具有挑战性的AppWorld和OfficeBench基准上评估了该方法，这些基准包含信息检索和状态变更两类任务。在合成数据上微调模型带来了显著的性能提升，证明无需任何可执行环境即可为API调用智能体生成有效的监督信号。我们的研究结果确立了基于LLM的API模拟是一种实用且可扩展的方案，可用于跨多种API生态系统的智能体训练。

## Abstract
Training API-calling large language model (LLM) agents demands massive amounts of high-quality trajectories. However, collecting such data at scale typically requires fully implemented environments with executable APIs and realistic, pre-populated backend databases, creating a major bottleneck for scalability. To overcome this, we propose an environment-free synthetic data generation approach that leverages LLMs as on-the-fly digital world models. Given only API specifications, our method generates trajectories mimicking interactions between an agent and a stateful environment. Specifically, an LLM first generates diverse tasks solvable with the provided APIs. A teacher agent then iteratively solves each task while an LLM simulator generates coherent synthetic API responses conditioned on the task context and simulation history. Finally, an LLM judge filters the trajectories to ensure the quality of the resulting dataset. We evaluate our approach on the challenging AppWorld and OfficeBench benchmarks, which include both information-retrieval and state-changing tasks. Fine-tuning models on our synthetic data yields significant performance gains, demonstrating that effective supervision for API-calling agents can be generated without any executable environment. Our results establish LLM-based API simulation as a practical, scalable solution for training agents across diverse API ecosystems.

### 2. 论文提出的方法论

- **核心框架：ESAT (Environment-Free Synthetic Agentic Trajectory generation)**
  - ESAT 是一个仅需 API 规范作为输入的三阶段合成数据生成流水线。

- **（一）任务合成（Task Synthesis）**
  - **分桶生成**：为确保任务多样性，基于难度、动作类型、任务焦点、应用数量等属性组合成一个**配置分桶网格**。任务生成器LLM在随机化的分桶序列中生成候选任务。
  - **逆频率采样**：为防止生成器只关注少数API，维护API调用频率计数，并以**反比于当前使用频率**的概率采样API，强制生成器关注长尾API。
  - **任务过滤与重写**：生成的候选任务首先由一个LLM评判器过滤，剔除不规范、不可解的任务。之后，通过LLM重写器将冗长、步骤化描述的初始任务**压缩成简洁、意图明确的用户请求**，最后再由评判器验证。

- **（二）轨迹合成（Trajectory Synthesis）**
  - **核心机制：LLM模拟器**
    - **交互模式**：一个充当教师的LLM智能体迭代地发出API调用；一个LLM模拟器扮演动态的数字世界模型，为每个API调用**即时生成**合成响应。
    - **上下文工程**：为确保响应的状态连贯性，模拟器的输入上下文包含：API规范、调用参数、待解决的任务、**先前同应用内的API调用及响应历史**、虚拟用户信息、当前日期时间。
  - **模拟器规则与质量检查**
    - LLM模拟器被四大规则约束：**数据一致性（维护跨轮次状态）、数据多样性（生成无关项以模拟真实噪声）、数据真实性（值符合字段约束）、模式合规**。
    - 响应经过多层验证：先通过程序化校验检测参数缺失、类型不匹配等问题；再通过程序化校验响应JSON是否符合输出模式；最后通过一个LLM评判器检查**语义正确性和与历史的一致性**。失败时进行重新生成，直至通过或重试次数耗尽。

- **（三）轨迹过滤（Trajectory Filtering）**
  - 最终生成并完成的轨迹会由一个轨迹级LLM裁判进行评估，判断是否正确解决了任务。一条轨迹会经过**多次评判**，仅保留**多数票为正面**的高质量轨迹加入训练集。

### 3. 实验设计

- **评估基准**
  - **AppWorld**：一个复杂多应用基准，包含457个API分布在9个应用中，要求信息检索和状态更改。分为Test-Normal（168个任务）和Test-Challenge（417个任务，包含2个未见过的应用）。
  - **OfficeBench**：一个办公自动化基准，包含20个API分布在8个应用中，专注于跨应用的长程工作流规划。

- **对比方法/数据**
  - **零样本（Zero-shot）**：原始基础模型的性能。
  - **AWT**：使用教师模型在真实的AppWorld环境中运行训练集任务所产生的轨迹数据进行训练。
  - **现有合成数据集**：与ToolAlpaca、ToolACE、Nemotron等已有的工具调用合成数据集进行了比较。
  - **其他方法**：在OfficeBench上与Simia方法进行了比较。

- **训练与评估细节**
  - **模型**：在Qwen3（1.7B-14B）和Qwen3.5（2B-27B）系列等多个尺寸的模型上进行监督微调（SFT）。
  - **评估指标**：AppWorld上使用任务目标完成率（TGC）和场景目标完成率（SGC）；OfficeBench上使用任务通过率。
  - **训练方式**：每个轨迹被格式化为多轮对话，模型在每一步预测智能体的输出，条件化于任务指令和之前的所有交互历史。结果报告为8次推理运行的平均值。

  - **轨迹合成**：使用 GLM-5.1-FP8。

### 6. 论文的主要结论与发现

- **环境无关数据的高效性**：ESAT生成的合成数据能赋予API调用智能体强大的能力，**在AppWorld上，用ESAT数据训练甚至优于使用真实环境轨迹数据（AWT）训练**。
- **强大的泛化能力**：仅用完全独立于目标环境的**合成应用API（ESAT-S52）** 生成的训练数据，就能使模型在未知的AppWorld API上展现出巨大性能提升，证明了方法传授的是通用的API调用技能。
- **有效的知识蒸馏**：用ESAT数据微调的小模型性能可**超越参数量远大于其的零样本前沿模型**（如GPT-4o、Nemotron-3-120B）。
- **高保真模拟**：对模拟器质量的独立分析表明，其产生的响应**93.7%都是有效且一致的**，验证了LLM作为数字世界模型的可行性。
- **优于现有合成数据方法**：与ToolAlpaca、ToolACE等不考虑有状态环境的工具调用数据集相比，ESAT提供的数据在复杂基准上带来了显著增益，证明了其为交互式、有状态场景生成数据的关键价值。

### 7. 优点

- **核心创新性与可扩展性**：彻底摒弃了对可执行环境的依赖，将数据生成简化为仅需API规范的轻量级问题，极大降低了扩展门槛。
- **方法论坚实**：三阶段流水线（合成、交互、过滤）设计周详，尤其“LLM作为有状态世界模型”的思想和实现（上下文管理、多层次验证）是杰出的工程与学术贡献。
- **实验论证极具说服力**：在两个高难度基准上、多模型尺寸下，对比了真实环境数据和多种现有合成数据方法，提供了全面且令人信服的证据。
- **数据多样性与覆盖度高**：“分桶生成”和“逆频率采样”策略确保了生成任务和轨迹在难度、类型及API覆盖上的平衡和广泛性。

### 8. 不足与局限

- **模拟器保真度衰减**：虽然整体质量高，但模拟器的失败率会随**输出长度增加而显著上升**（超过2K tokens时失败率达23%），表明LLM在生成长篇且有状态的响应方面存在固有限制，这可能影响长程、复杂任务的数据质量。
- **对前沿LLM的依赖**：生成高质量合成数据依赖于强大的LLM（如GLM-4.7/5.1、Gemini-3.1-Pro），虽然成本可能低于构建真实环境，但仍对内部或API服务有较强依赖。
- **多应用任务生成难题**：尽管有分桶策略，生成的任务在应用数量分布上仍偏斜向单应用任务，多应用（2-3个）的复杂工作流任务占比较低，可能影响模型处理跨应用协作的能力。
- **与现实环境的偏差**：LLM模拟的数字世界模型无论多么逼真，终究是真实世界的统计近似，可能无法完美捕捉真实API中所有边缘情况、错误模式和随机性。
- **仅限SFT训练**：论文仅验证了生成数据用于监督微调的效果，未探索其在强化学习等其他训练范式下的潜力。

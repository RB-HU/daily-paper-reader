## 摘要
针对美国证券交易委员会（SEC）文件的金融问答需要检索和综合分散在冗长、标准化且高度冗余披露中的异构证据。现有的检索增强和多智能体系统通常直接从用户问题中派生检索查询，并按语义相似度对候选结果排序。这些选择共同导致了先验语料不对齐：即模型先验与目标文件结构、术语和证据标准之间的不匹配。因此，查询生成遗漏了语料特定的证据，而语义重排序则倾向于选择主题相似但证据无效的假阳性片段。我们提出了FinSAgent，一个证据支撑的多智能体框架，该框架将SEC文件问答重新定义为语料对齐的检索规划，并通过一个简单原则纠正两端：在模型先验可能占主导之处注入语料侧条件。FinSAgent结合了（1）锚定于10-K规定项目结构的角色专用智能体，（2）基于数据库感知的查询分解，使每个智能体的子查询以轻量级的摘要级别局部语料视图为条件，以及（3）利用学习的特征门控重排序器进行多路径检索，将证据有效性与语义相似性分离开来。在五个离线金融问答基准测试中，FinSAgent在检索覆盖率和答案正确性上均优于强大的单智能体和多智能体基线；在一项包含1000名匿名用户评分的三臂随机在线实验中，它也比基线获得了更高的评分。

## Abstract
Financial question answering over U.S. Securities and Exchange Commission (SEC) filings requires retrieving and synthesizing heterogeneous evidence dispersed across long, standardized, and highly redundant disclosures. Existing retrieval-augmented and multi-agent systems typically derive retrieval queries directly from the user's question and rank candidates by semantic similarity. Together, these choices create prior-corpus misalignment: a mismatch between model priors and the target filings' structure, terminology, and evidence standards. As a result, query generation misses corpus-specific evidence, while semantic reranking favors topically similar but evidentially invalid false-positive chunks. We propose FinSAgent, an evidence-grounded multi-agent framework that reframes SEC filing QA as corpus-aligned retrieval planning and corrects both ends with a single principle: inject corpus-side conditioning wherever model priors would otherwise dominate. FinSAgent combines (1) role-specialized agents anchored to the mandated 10-K item structure, (2) database-aware query decomposition that conditions each agent's sub-queries on a lightweight, summary-level view of the local corpus, and (3) multi-path retrieval with a learned feature-gated reranker that separates evidential validity from semantic similarity. Across five offline financial QA benchmarks, FinSAgent improves retrieval coverage and answer correctness over strong single-agent and multi-agent baselines; in a three-arm randomized online experiment with 1,000 anonymous user ratings, it also receives higher scores than baselines.

*   **现有方法的根本缺陷**：论文指出现有RAG和多智能体系统存在**先验-语料错配问题**。即模型基于自身训练知识（先验）直接生成查询和排序，而非依据具体文件的结构、术语和证据标准。

因此，论文的核心目标是**纠正这种错配**，通过注入语料侧信息来引导检索和证据筛选，实现“证据支撑”的答案生成。

## 2. 方法论：FinSAgent框架

FinSAgent的核心设计原则是：**在任何模型先验可能占主导地位的环节，注入语料侧的条件信息**。它通过一个三阶段流水线实现：

1.  **阶段一：角色专用的并行多智能体（纠正检索前的覆盖面）**
    *   **核心思想**：将问题的不同证据需求解耦，分配给锚定于10-K文件法定结构的角色专用智能体，以扩大检索覆盖面。
    *   **技术细节**：
        *   设计了五个智能体角色，每个对应10-K中的特定项目：**通用、定量、市场、法务、公司**。
        *   每个智能体`a`拥有一个角色策略（职责、检索范围、分析焦点），并行地将用户问题`q`转化为一个角色条件化的分析视图`z_a`。

2.  **阶段二：数据库感知的查询分解（纠正查询端的错配）**
    *   **核心思想**：解决LLM在生成子查询时仅依赖内部先验的问题，让其“看到”本地语料库的概貌。
    *   **技术细节**：
        *   **离线摘要**：预先为文档的连续片段生成紧凑摘要，并建立索引。
        *   **运行时条件注入**：在查询时，先用轻量级检索器FAISS从摘要索引中获取与问题最相关的`k0`个摘要，形成**本地数据库视图**。然后，每个智能体基于其分析视图`z_a`和这个数据库视图，通过提示LLM来生成一系列语料对齐的角色专用子查询`Q_a`。
        *   **优势**：使生成的子查询能反映文件的章节标题、细分市场名称和披露术语。

3.  **阶段三：多路径检索与特征门控重排序（纠正证据选择端的错配）**
    *   **多路径检索**：为每个子查询，从BM25（稀疏）、FAISS（密集向量）、摘要级等不同路径并行检索，取并集构成高召回率的候选集`C_{a,j}`。
    *   **特征门控重排序**：
        *   **核心思想**：解决语义重排序器无法区分的“假阳性”片段问题。它使用一个学习到的模型来评估证据的**非语义有效性**。
        *   **技术细节**：
            *   为每个候选项创建一个包含31个非语义特征的向量（如路径来源、排名、词汇重叠、页码等）。
            *   使用一个**LightGBM模型**`M`学习估计候选项的有效性概率`p`。
            *   **乘法门控机制**：最终的得分公式为 `s_{final} = s_{rr} * (1 - λ * (1 - p))`。其中`s_{rr}`是语义重排序分数，`λ`是门控强度。这有效地对“语义高分但证据高风险”的候选项进行降权。
    *   **答案合成**：各智能体基于筛选后的证据生成子答案，最后由编排器整合出最终答案。

## 8. 不足与局限

*   **数值推理精度**：即使在FinSAgent解决了检索和证据融合问题后，**数值精度错误（C1类错误）仍为主要剩余瓶颈**，这说明框架的后续环节（如复杂计算）仍需加强。

*   **工程复杂性**：多智能体协调、索引、多路径检索、门控模型训练等模块显著增加了系统的工程复杂性和运维成本。

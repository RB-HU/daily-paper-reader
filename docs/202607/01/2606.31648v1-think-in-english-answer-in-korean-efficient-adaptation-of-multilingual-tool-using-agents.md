## 摘要
我们提出了 LuckyStar 111B，这是一个 1110 亿参数的混合推理模型，由 Cohere 与 LG CNS 合作开发，用于在实用内存和服务约束下构建韩英企业智能体。该模型基于 Cohere 已完全后训练的 Command A 模型进行训练，而非从头预训练，并利用前导条件在简洁的非推理行为与更长的工具导向推理之间切换。我们研究了高效扩展工具使用智能体的四种选择：多语言监督微调、在多步工具使用任务中采用带可验证奖励的强化学习、针对面向韩语用户的响应添加语言一致性奖励，以及用于单 GPU 服务的 4 位量化。适应后的模型在数学推理、函数调用和代理式自然语言转 SQL（NL2SQL）性能上均有提升，同时保持了通用的韩语和英语指令遵循质量。这些结果为在内存受限部署下将后训练多语言模型适应于可验证代理工作流提供了实用方法和失效模式分析。

## Abstract
We present LuckyStar 111B, a 111B-parameter hybrid reasoning model developed through a collaboration between Cohere and LG CNS for Korean-English enterprise agents under practical memory and serving constraints. The model trains from Cohere's fully post-trained Command A model rather than a new pretraining run, and uses preamble conditioning to switch between concise non-reasoning behavior and longer tool-oriented reasoning. We study four choices for scaling tool-using agents efficiently: multilingual supervised fine-tuning, reinforcement learning with verifiable rewards for multi-step tool-use tasks, language-consistency rewards for Korean user-facing responses, and 4-bit quantization for single-GPU serving. The adapted model improves mathematical reasoning, function calling, and agentic natural-language-to-SQL (NL2SQL) performance while preserving general Korean and English instruction-following quality. These results provide a practical recipe and failure-mode analysis for adapting post-trained multilingual models to verifiable agentic workflows under memory-constrained deployment.

## 1. 论文的核心问题与整体含义

- **核心问题**：如何在严格的内存与服务约束下，将一个已后训练的大规模多语言模型高效地适配为能够进行复杂推理与工具调用的企业级智能体，同时保持其多语言指令遵循能力。
- **研究背景与动机**：
  - 企业助手需在私域数据上推理、调用工具并作答，且实际部署存在GPU显存、推理带宽等硬件限制。
  - 多语言场景（如韩语-英语）增加了难度：用户以韩语提问，模型需解析英语为主的数据库模式或文档，执行SQL或检索工具，最终以韩语回复。
  - 从零预训练成本高昂，因此从已有后训练模型出发，添加推理与工具使用行为是一条更经济的路径。

## 2. 论文提出的方法论

- **总体框架**：LuckyStar 111B 以 Cohere 的 Command A（111B参数，已完全后训练）为基座，通过三阶段管线进行适配，并使用前导条件（preamble conditioning）让同一份权重支持两种行为模式。
- **关键技术细节**：
  - **混合监督微调**：
    - 约80%的样本为推理示例（数学、代码、工具调用），20%为非推理的指令遵循样本，从而保留基座模型的通用多语言能力。
    - 推理示例附加前导文本，引导模型生成逐步思考后再给出最终答案；非推理示例使用简洁回答的前导。
    - 对于仅有最终答案的地面真相，采用拒绝采样与LLM评判生成正确推理轨迹。
  - **多语言推理策略**：“用英语思考，用韩语作答”
    - 实验发现，即使提示为韩语，用英语生成的推理轨迹质量更高（如AIME 2024 69.3 vs 50.9）。
    - 通过拒绝采样生成2万条“英语推理链条 + 韩语最终答案”的示例加入SFT，将英语推理与用户语言输出解耦。
  - **可验证奖励强化学习**：
    - 使用REINFORCE Leave-One-Out（RLOO）算法，对数学和NL2SQL任务设定二元奖励。
    - 对韩语提示，引入语言一致性惩罚：
      
      \[
      r = r_{\text{correctness}} - r_{\text{penalty}}, \quad r_{\text{penalty}} = 
      \begin{cases}
      0.5, & \text{若最终答案语言不匹配} \\
      0,   & \text{其他}
      \end{cases}
      \]
    - 训练时过滤超长（>32k token）的轨迹而非直接给零奖励，避免模型学会过早终止。
  - **冷启动处理**：针对NL2SQL任务基座策略准确率不足1%的问题，先用强模型通过最佳-N拒绝采样生成正确的工具调用轨迹并加入SFT混合。
  - **离线偏好对齐（DPO）**：以低学习率微调，修正RLVR后模型过于冗长的问题，偏好直接回答，拒绝啰嗦输出。
  - **4位量化**：将FP8权重压缩为4位，降低约50%内存占用，使111B模型可在单张80GB H100上服务。

## 6. 论文的主要结论与发现

- 通过**混合SFT + 可验证RL + DPO**的三阶段适配，可以大幅提升后训练模型的数学推理和代理式工具使用（如企业NL2SQL从7.3提升至38.0，LG评价从2.67提升至4.85），同时保持通用多语言指令遵循质量基本不变。
- **“英语推理+韩语作答”**是一种务实的多语言推理方案，配合语言一致性奖励可有效抑制最终答案向英语漂移。
- 4位量化在多数基准上保持了与FP8相当的精度，使得千亿级参数模型能在单H100上服务，降低了企业部署成本。
- 指出了三类关键失效模式：NL2SQL冷启动、语言漂移、RL后冗长倾向，并给出了针对性的缓解措施。

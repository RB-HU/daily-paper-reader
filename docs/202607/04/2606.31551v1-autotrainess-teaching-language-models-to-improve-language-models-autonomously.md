## 摘要
训练语言模型（LMs）仍是一个高度人力密集的过程，即使前沿的语言模型代理在软件工程和其他长周期任务中变得越来越有能力。一个核心挑战是，自主后训练不仅仅是一个编码问题：它要求代理反复规划迭代、构建与基准对齐的数据、运行稳定的训练作业、评估检查点，并在数小时的交互中保持实验状态。我们提出了AutoTrainess，一个语言模型代理，将这些操作暴露为一组用于规划、数据准备、训练、评估和日志记录的代理-计算机接口。AutoTrainess并不是让代理在缺乏明确动作空间的原始CLI环境中操作，而是将先前的人类经验外化为明确的工作流、规则和执行约束，引导代理实现有效和可靠的训练行为。在PostTrainBench上，AutoTrainess始终优于仅使用CLI的基线，使用GPT-5.4（Codex）时平均得分26.94，而仅使用CLI的基线为23.21。它还能跨模型和框架泛化，将DeepSeek-V4-Flash（OpenCode）的得分从12.13提升到19.58。

## Abstract
Training language models (LMs) remains a highly human-intensive process, even as frontier language model agents become increasingly capable at software engineering and other long-horizon tasks. A central challenge is that autonomous post-training is not just a coding problem: it requires the agent to repeatedly plan iterations, construct benchmark-aligned data, run stable training jobs, evaluate checkpoints, and preserve experiment state across many hours of interaction. We present AutoTrainess, a LM agent that exposes these operations as a repository of agent-computer interfaces for planning, data preparation, training, evaluation, and logging. Rather than leaving the agent to operate in a raw CLI environment with an underspecified action space, AutoTrainess externalizes prior human experience as explicit workflows, rules, and execution constraints that guide the agent toward effective and reliable training behavior. On PostTrainBench, AutoTrainess consistently outperforms CLI-only baselines, achieving 26.94 average score with GPT-5.4 (Codex) versus 23.21 for CLI-only. It also generalizes across models and harnesses, improving DeepSeek-V4-Flash (OpenCode) from 12.13 to 19.58.

### **论文提出的方法论**
*   **核心思想**：提出 **AutoTrainess**，一个通过将人类的先前经验**外化**为一组特定于训练的**代理-计算机接口（ACI）** 来赋能LM代理的系统。其核心理念是让代理受益于人类研究和工程实践中的结构化知识、工作流和约束，而非凭空解决训练问题。
*   **关键技术细节**：AutoTrainess的ACI系统被称为 **AutoTrainHub**，它将自主后训练组织成一个包含四个模块的闭环工作流，每个模块都封装了相应的规则和最佳实践：
    *   **数据处理模块**：提供三种显式动作来构建与任务对齐的训练数据。
        1.  **数据选择**：根据之前的失败模式，确定数据需求并选择初始数据源（如本地、外部或模型蒸馏数据）。
        2.  **数据构建**：支持抽取、清洗、去重、重写、合成、蒸馏、模式规范化等操作，并要求代理在构建前检查基准的评估接口。
        3.  **数据验证**：检查数据集的格式、基准对齐、内容质量和泄漏风险，并明确返回“批准”、“返回构建”或“返回选择”三种结果，以区分执行层面的错误与方向性的错误。
    *   **训练模块**：提供一个基于 **LlamaFactory** 的稳定训练入口。代理被约束使用此固定后端，进行全参数微调，并在大规模运行前必须进行小批量验证。对于强化学习，则要求明确定义奖励信号。
    *   **评估模块**：使用基准任务的标准评估流程对检查点进行测试，生成包含15个随机样本及其得分的详细摘要，并要求代理总结主要失败模式，并将其归类为数据、训练或推理/模板问题，为下一步迭代提供结构化诊断。
    *   **日志记录与规划模块**：
        *   **日志记录**：在每次迭代后向实验日志追加一个结构化条目，记录迭代背景、动机、方法、结果、分析和下一步行动，作为贯穿整个训练过程的长期记忆。
        *   **规划**：要求代理基于过往实验证据，明确当前迭代的主要目标、计划干预措施和具体的成功标准。
*   **公式或算法流程**：AutoTrainess 的本质不是提出一个新的算法或公式，而是将自主训练过程建模为一个**结构化的顺序决策过程**。其流程如下：代理首先通过`规划`接口根据历史证据制定目标，然后进入`数据`接口进行数据的选择、构建和验证，传递有效数据至`训练`接口执行标准化训练，再将结果模型送入`评估`接口获取性能反馈，最后通过`日志`接口记录完整迭代信息，并回到`规划`接口开始下一轮循环。

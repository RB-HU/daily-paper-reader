## 摘要
大型语言模型（LLMs）在高风险场景下的说服任务中经常遇到困难。人们的个体性格与关切需要量身定制的策略，而非一刀切的方法。为应对这一挑战，我们聚焦于火灾救援场景，其中接线员必须说服居民撤离，以此作为高风险说服领域，并提出了对话策略选择（DiPS），一个基于Q学习的框架，用于动态选择适应不断变化的对话语境的说服策略。具体而言，我们训练一个评论家，以最大化撤离成功率为目标，在每一轮根据居民最近的发言选择一种说服策略。然后，我们在模拟交互和真实人类交互中，将DiPS与多个基线进行了比较。我们发现，DiPS比零样本LLM和通用RAG增强方法取得了更高的撤离成功率。

## Abstract
Large Language Models (LLMs) often struggle with persuasion in high-stakes scenarios. People's individual personalities and concerns require tailored strategies rather than a one-size-fits-all approach. To address this challenge, we focus on a fire-rescue scenario in which an operator must persuade a resident to evacuate as a high-stakes persuasion domain and propose Dialogue Policy Selection (DiPS), a Q-learning framework to dynamically select persuasion strategies adapted to the evolving conversational context. Specifically, we train a critic, trained to maximize the chance of evacuation success, to select a persuasion policy at each turn based on the resident's recent utterances.We then evaluate DiPS against multiple baselines in both simulated and real human interactions. We find that DiPS achieves higher evacuation success than a zero-shot LLM and generic RAG-augmented approach.

- **整体含义**: 论文提出了一种基于强化学习的对话策略选择框架 **DiPS**，它将说服视为一个部分可观测的马尔可夫决策过程，在每一次对话轮次中，利用训练好的 Q 函数从一组候选策略（人格）中选出最优策略，从而驱动 LLM 生成更贴合当下情境的回应。这一工作将**离线 Q 学习**引入高风险说服对话，证明了动态策略级决策比较静态提示或全局检索更有效。

## 2. 方法论：DiPS 框架
### 2.1 核心思想
- 将交互建模为 **POMDP**，其中居民内部状态不可直接观测，接线员只能根据居民的发言（观测）和对话历史做出回应。
- 不直接优化自然语言生成，而是在**策略层面**进行选择：维护一组预定义的离散说服策略（personas），例如“共情型”“指令型”“信息型”等，每轮由 Q 函数选择最合适的策略，然后用该策略条件化 LLM 生成操作员话语。

### 2.2 关键技术细节
- **状态表示**: 将最近若干轮居民的发言通过 MiniLM 编码为句子嵌入，作为当前状态 $s_t$。
- **动作空间**: 离散的动作 $z_t$ 对应选择一个策略（persona）索引。
- **奖励**: 稀疏奖励，仅在对话最终成功（居民同意撤离）时给予 $r=1$ ，否则为 $0$。
- **训练算法**: 使用 **Implicit Q-Learning (IQL)**，这是一种离线强化学习方法，适合处理分布偏移，避免对未见动作的高估。通过期望回归学习状态价值函数，再用优势加权回归提取策略。
- **离线数据集构建**: 利用 Wizard-of-Oz 收集的人类操作员对话数据，按不同操作员风格划分出多种策略，每条对话轨迹转换为 $(s_t, z_t, r_t, s_{t+1})$ 元组，只使用训练集居民的对话训练 Q 函数。
- **推理时**: 计算 $z_t = \arg\max_{z} Q(h_t, z)$ ，然后用选中的策略及其对应的检索示例（RAG）和策略描述提示 LLM 生成回复。

### 2.3 算法流程（文字描述）
1. 利用对话历史（最近居民发言）生成状态嵌入。
2. 通过预训练的 IQL Q 函数计算每个可选策略的期望价值。
3. 选择价值最高的策略。
4. 将该策略的描述文本和相关检索示例填入操作员 prompt，调用 LLM 生成回复。
5. 居民（模拟器或真人）回应，更新历史，循环直至对话结束或达到最大轮次。

## 3. 实验设计
### 3.1 场景与角色
- **领域**: 山火疏散说服，操作员与居民进行多轮文本对话，目标是说服居民撤离。
- **居民角色**: 共 10 个，其中 5 个来自已有 Wizard-of-Oz 数据（Bob, Niki, Lindsay, Michelle, Ross），5 个为新创建角色（Mary, Ben, Ana, Tom, Mia），每个具有不同的性格和关切（如顽固、照顾孩子、需要车辆等）。

### 3.2 基准方法对比
- **Zero-shot**: LLM 直接作为操作员，无任何历史数据或策略信息。
- **RAG (Global Retrieval)**: 从所有成功操作员话语中检索示例增强 prompt，但不区分策略。
- **DiPS (IQL + Per-Policy RAG)**: 本方法，IQL 选策略，每个策略带专属 RAG 检索。
- 改进模拟中增加了更多消融：
  - 随机策略选择（全量/子集、带/不带 RAG 和策略描述）
  - 不匹配策略（用固定但与居民不匹配的策略）
  - DiPS 的组件消融：取消每策略检索改为全局检索、仅保留策略描述不检索、仅检索不提供策略描述等。

### 3.3 评估方式
- **实验 1 (AI 模拟居民)**: 用 GPT-4o-mini 扮演居民，每种方法每个居民运行 3 次，共 90 次对话。用 LLM-as-judge 和人工标注评判成功与否。
- **实验 2 (人类受试者)**: 招募真人扮演居民，每个参与者与所有三种方法随机顺序交互，事后报告是否被说服撤离，并评价连贯性。
- **实验 3 (改进模拟)**: 根据前两个实验的差异，提升居民模拟器真实度（加入明确障碍信息），并增设 10 种对比方法（包含更多随机和消融条件），每个居民每种方法运行 5 次，重点关注成功率与平均轮次。

## 4. 资源与算力
- **模型**: 模拟中使用的骨干 LLM 为 **GPT-4o-mini** (通过 API 调用)，操作员和居民模拟器均基于此模型。
- **强化学习训练**: 状态编码使用 MiniLM 句子嵌入（轻量），动作空间仅数个离散策略，训练 IQL 不依赖 GPU 集群。论文**未明确提及**训练所用的 GPU 型号、数量、训练时长等算力细节。

- **实验组数**:

  - 实验 2：人类受试者，总共 78 次交互（具体分配见表 1），涉及三种方法。

- **公平性**: 所有方法共用相同的对话引擎和 LLM，区别仅在于 prompt 和策略选择方式，对比条件控制较为严格。LLM-as-judge 与人工标注一致性报告为 72.2%（Cohen’s κ 0.42），对于困难场景的可解释性尚可。

## 6. 主要结论与发现
- 在最初的 LLM 模拟中，DiPS 并未表现出优势（甚至弱于零样本），但**在人类受试者实验中，DiPS 获得最高的总体成功率（79%）**，且对训练集中和新见居民均有效。
- 改进模拟环境后（更真实地反映居民特有的顽固性和依赖条件），趋势与人类实验一致：**DiPS 以 92% 的成功率超过所有其他方法**，且平均对话轮次更短（11.2 轮），说明其不仅更有效，也更高效。
- 消融实验表明，**策略选择、每策略检索和策略提示三者缺一不可**，移除任一部分都会导致成功率下降和对话延长。
- 离线的 IQL 能够从异质的对话数据中学习到有意义的轮次级策略切换模式，而简单的随机策略或固定策略效果显著变差。
- 评估环境对结论影响很大：没有正确模拟居民必要需求条件的模拟器会掩盖自适应方法的优势。

## 7. 优点
- **方法创新**: 将离线 Q 学习应用于高风险多轮说服的场景，且选择动作作用于**策略层面**而非直接生成文本，既保留了 LLM 的生成灵活性，又实现了可控的策略级决策。
- **实验严谨**: 通过模拟 + 人类 + 改进模拟的三阶段验证，揭示了模拟与现实的差距，强调真实环境模拟的重要性。
- **解释性强**: 提供了离散策略的切换解释，有助于理解模型在不同阶段选择何种说服风格。
- **轻量训练**: 仅依赖离线对话数据和句子嵌入，无需复杂的在线环境或大规模训练，易于复现。
- **与人类操作员对比**: 将 LLM 代理的性能与原始人类接线员数据进行对比，指出了当前代理在需要具体协调和行动承诺的案例中仍然不足。

## 8. 不足与局限
- **策略集固定**: DiPS 只能从预定义的有限策略中选择，可能遗漏更细微或创造性的回应方式。
- **奖励稀疏**: 仅终端成功给予奖励，可能导致中间轮的信用分配困难，对学习有效的局部策略有挑战。
- **依赖离线数据**: 如果某些类型的居民行为或失败模式在离线数据中不充分，Q 函数可能无法泛化。
- **模拟差距**: 即便改进后的模拟，仍不能完全代表真实人类在面对生命威胁时的复杂反应，可能导致高估或低估。
- **单语言局限**: 所有实验仅基于英语 Wizard-of-Oz 数据，未验证在多语言环境下的表现。
- **LLM 裁判局限性**: 用来替代人工评估的 LLM-as-judge 虽有一致性但趋于保守，可能影响自动评估的准确性。
- **应用限制**: 仅测试了火灾撤离场景，泛化到其他高风险说服领域（如医疗、金融等）需要进一步验证。

## 摘要
支付、电子商务和电信系统中的欺诈检测要求在个体层面具有准确性，在严重类别不平衡下具有鲁棒性，并且便于风险管理者理解。现有方法至少不满足其中一项要求：自动化机器学习系统在固定的数值空间中搜索，缺乏对数据集的语义感知；基于图神经网络的方法需要预定义的关系图，并且在个体决策层面不透明；而通用大语言模型（LLM）代理的设计没有考虑现实世界欺诈检测特有的召回率和精确度约束。在本文中，我们提出了SAGE，这是第一个由LLM驱动的端到端多代理框架，用于欺诈检测。SAGE协调三个专用代理，它们基于六层数据诊断树（DDT）和由自然语言梯度引导的马尔可夫决策过程做出决策，在欺诈特定的奖励下自动优化模型。在五个欺诈数据集和五个LLM主干上，SAGE在96.00%的方法—数据集比较中获胜，F1值平均比基线提高了40.86%。代码可在https://github.com/yichenC1c/SAGE获取。

## Abstract
Fraud detection in payment, e-commerce, and telecommunications systems requires accuracy at the individual level, robustness under severe class imbalance, and ease of understanding for risk managers. Existing methods fall at least one of these requirements: automated machine learning systems search a fixed numerical space without semantic awareness of the dataset; graph neural network-based methods require pre-defined relational graphs and remain opaque at the individual-decision level; and the design of general-purpose large language model (LLM) agents does not consider the recall and precision constraints specific to real-world fraud detection. In this paper, we propose SAGE, the first end-to-end LLM-driven multi-agent framework for fraud detection. SAGE coordinates three dedicated agents that make decisions based on a six-layer Data Diagnostic Tree (DDT) and a Markov decision process guided by natural-language gradients, automatically optimizing the model under a fraud-specific reward. On five fraud datasets and five LLM backbones, SAGE wins $96.00\%$ of method--dataset comparisons and improves F1 by an average of $40.86\%$ over baselines. The code is available at https://github.com/yichenC1c/SAGE.

## 方法论

SAGE 是首个面向表格数据欺诈检测的端到端 LLM 驱动多代理框架，包含三个严格顺序执行的专用代理，并融合**六层数据诊断树（DDT）**与**自然语言梯度驱动的有限时域马尔可夫决策过程（MDP）**。

- **总体映射关系**  
  SAGE 可视为复合映射：\( \text{SAGE}: \mathcal{P} \to \mathcal{C}\times\mathcal{M}\times\mathbb{R}^d\)，输入为欺诈检测问题实例 \(P\)，输出为最优代码 \(c^*\)、最佳模型 \(M^*\) 和对应的四维指标向量（含 F1、AUPRC、MCC、R@FPR 10⁻⁴）。该映射由三个代理 \(A_1, A_2, A_3\) 级联实现。

- **代理一：Profiling Agent (\(A_1\)) —— 构建数据诊断树（DDT）**  
  输入原始数据集，输出统计报告与 DDT。DDT 包含六个语义层，通过规则化统计和一次 LLM 语义解释调用，将高维数据压缩为约 2,000 tokens 的结构化树形文本，解决长上下文中的信息过载和注意力分散问题。各层内容为：
  - **Lscale**：行数、特征数、内存占用。
  - **Llabel**：欺诈率、不平衡比、正负样本计数。
  - **Lfeature**：数值/类别特征计数、与标签最相关的特征。
  - **Lquality**：缺失值统计、高缺失率特征。
  - **Lstructure**：ID 型列、时间列、子集欺诈特征。
  - **Ldiagnosis**：LLM 推理出数据集类型、关键风险特征、适用于奖励函数的召回率阈值 \(\tau\)。

- **代理二：Planning Agent (\(A_2\)) —— 算法选择与初始代码合成**  
  接收 DDT 和报告，由 LLM 自主决定算法（如 XGBoost、LightGBM 等），生成包含数据加载、预处理、特征工程、训练与评估的固定五段式初始代码 \(c_0\)，并执行得到初始模型 \(M_0\) 和基线指标 \(m_0\)。代码中预先标注“# handle:”注释，为后续优化提供编辑锚点。

- **代理三：Optimization Agent (\(A_3\)) —— 自然语言梯度 MDP 优化**  
  将代码迭代优化建模为一个有限时域 MDP：
  - **状态** \(s_t = (c_t, m_t, H_t)\)：当前代码、验证集指标、历史轨迹（包含过去的动作、奖励、自然语言诊断）。
  - **动作空间** 包含七种可编辑句柄：缺失策略、特征变换、正则化、树深度/基估计器数、采样参数、阈值调优、算法切换。
  - **优化循环** 每轮执行两步：
    1. **评论家调用** LLM_critique：基于当前状态输出**自然语言诊断** \(D_t\)（相当于文本梯度），指出瓶颈和改进方向。
    2. **执行者调用** LLM_codegen：将 \(D_t\) 与选定动作转换为具体代码编辑。
  - **收敛条件**：达到最大迭代次数 \(K=20\)，或连续 \(k=4\) 轮奖励提升小于 \(\varepsilon=0.001\)。

- **复合奖励函数**  
  奖励 \(R(m_t)\) 由四个加权指标（F1、AUPRC、R@FPR 10⁻⁴、召回率）的加权和，乘以两个闸门因子（\(W_{recall}\) 惩罚低于阈值 \(\tau\) 的召回率，\(W_{precision}\) 防止退化为平凡高召回分类器），并加上基线提升奖励 \(B\)。该设计确保优化过程必须同时权衡检测质量与业务约束，禁止为提升 F1 而牺牲召回率。

- **复杂度与自治性**  
  总成本为 \(O(K(L+E))\)，其中 \(L\) 为单次 LLM 调用延迟，\(E\) 为单次沙箱训练耗时。用户仅需提供数据集与标签列，全流程无需人工干预。

## 主要结论与发现

- SAGE 在 **96%** 的方法–数据集比较中胜出，平均 F1 提升 **40.86%**，在极端不平衡的 PaySim 上 F1 达 0.9973，远优于崩溃的 RS（0.1593）。
- 经统计检验，SAGE 平均排名显著优于全部自动化基线，并达到甚至部分超越人类专家水平。
- SAGE 对底层 LLM 主干选择不敏感：跨 5 个主干的赢率均高于 85%（均值 88.8%），AUPRC 跨主干平均散布仅 1.13%，且跨数据分裂的标准差极小，展示了框架本身的鲁棒性。
- 自然语言梯度使每次迭代的决策（诊断→动作）完全可审计，复合奖励能做出真正的多目标权衡：宁可选择满足召回约束的低 F1 模型，也不选择高 F1 但违反约束的模型。
- 消融实验表明三个代理贡献互补，移除任一均导致 F1 显著下降（A3 贡献最大，降低 36.19%）。

## 优点

- **任务专用框架**：首次将 LLM 代理系统引入表格欺诈检测，并内置召回/精度业务约束，填补了 AutoML 缺少语义、GNN 依赖图结构、通用 LLM 代理不适用于欺诈领域的空白。
- **创新的 DDT 设计**：将极为冗长的高维表格数据压缩为语义丰富的六层树形结构，既解决 LLM 上下文窗口限制，又为下游决策提供可溯源的证据。
- **自然语言梯度 MDP**：通过“诊断—动作”闭环，在代码空间实现可解释的迭代优化，其自然语言记录直接构成审计轨迹，符合风控行业合规要求。
- **复合奖励设计**：通过惩罚性闸门和基线奖励，真实反映欺诈业务多目标且约束严格的评价标准。
- **实验全面严谨**：覆盖公开与工业数据，对比 AutoML、人类专家和 LLM 基线，五种子、多主干、统计显著性检验，且提供消融与案例研究，验证了方法的有效性和鲁棒性。
- **可私有化部署**：支持本地 LLM 推理，满足数据隐私场景需求。

## 不足与局限

- **未集成多模型优势**：SAGE 输出单一模型，在部分低维数据集上略逊于 AutoGluon 的集成堆叠，未能发挥混合模型架构的潜力。
- **忽略关系图信息**：当前仅利用个体表格特征，未融合 GNN 擅长的用户‑设备‑交易交互图结构，限制了在大规模关系化场景中的进一步收益。
- **实验覆盖局限**：评估限于表格型借贷、支付、电商、加密货币和电信欺诈，可能无法直接泛化至需要多模态信息（如文本、音频）的欺诈类型。
- **缺少计算开销的细粒度报告**：虽给出推断与训练的大致耗时，但未提供各数据集上完整的端到端墙钟时间与不同主干的详细成本对比。
- **受 LLM 推理能力影响**：尽管跨主干鲁棒，决策质量仍依赖于 LLM 的数据分析能力；若 LLM 对 DDT 理解出现偏差，可能影响下游代理的表现。

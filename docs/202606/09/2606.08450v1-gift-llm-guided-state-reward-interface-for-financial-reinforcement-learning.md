## 摘要
金融投资组合交易自然地表述为一个强化学习问题，其中智能体在变化的市场条件下依次重新平衡资产，以平衡收益、风险和交易成本。然而，在非平稳市场中，原始的OHLCV状态和短期收益奖励往往提供了一个不够明确的学习接口，这促使人们使用大语言模型将金融知识注入状态和奖励设计，同时约束开放式生成。为此，我们提出了GIFT，一个基于PPO的金融强化学习中状态-奖励接口设计的LLM引导框架。GIFT不是使用LLM做交易决策，而是使用因子引导的状态增强（Factor-guided State Enhancement）从金融因子基元生成状态特征，使用风险规则引导的奖励塑造（Risk-rule-guided Reward Shaping）从投资组合风险规则生成辅助奖励，以及使用诊断引导的精炼（Diagnostic-guided Refinement）利用PPO展开诊断修订候选接口。在精炼之后，GIFT在评估前固定所选的状态-奖励接口，测试时不再进行LLM查询或接口更新。跨越不同市场状态和投资组合场景的全面滚动窗口实验表明，GIFT相比基线改善了学习信号质量以及样本外风险调整后的投资组合表现。代码和数据可在以下网址获取：https://github.com/KAG778/GIFT。

## Abstract
Financial portfolio trading is naturally formulated as a reinforcement learning problem, where an agent sequentially rebalances assets under changing market conditions to balance return, risk, and transaction costs. Yet in non-stationary markets, raw OHLCV states and short-horizon return rewards often provide an under-specified learning interface, motivating large language models as a way to inject financial knowledge into state and reward design while constraining open-ended generation. To this end, we propose GIFT, an LLM-guided framework for state-reward interface design in PPO-based financial reinforcement learning. Rather than using the LLM to make trading decisions, GIFT uses Factor-guided State Enhancement to generate state features from financial-factor primitives, Risk-rule-guided Reward Shaping to generate auxiliary rewards from portfolio-risk rules, and Diagnostic-guided Refinement to revise candidate interfaces using PPO rollout diagnostics. After refinement, GIFT fixes the selected state-reward interface before evaluation, with no further LLM queries or interface updates at test time. Comprehensive rolling-window experiments across diverse market regimes and portfolio scenarios demonstrate that GIFT improves learning-signal quality and out-of-sample risk-adjusted portfolio performance over baselines. Code and data are available at: https://github.com/KAG778/GIFT .

### 2. 论文提出的方法论

GIFT是一个基于PPO的金融强化学习框架，通过三个核心组件构建更好的状态-奖励接口：

#### 2.1 因子引导的状态增强（Factor-guided State Enhancement, FSE）
- **核心思想**：从预定义金融因子库（如动量、波动率、下行风险、流动性、均值回归等）中，引导LLM选择、组合并生成额外的状态特征，附加到原始OHLCV输入上。
- **实现方式**：在每一轮精炼中，LLM根据市场统计信息和上一轮反馈，生成K个候选状态增强函数。这些函数对每个资产输出规范化的因子通道，并与原始紧凑价格编码拼接，形成增强的状态表示。
- **公式示意**：  
  \(\tilde{s}_{i}^{m,k} = [c(s_i^t) \parallel F_{m,k}(s_i^t)]\)，其中\(c(\cdot)\)为原始紧凑编码，\(F_{m,k}(\cdot)\)为生成的因子转换。

#### 2.2 风险规则引导的奖励塑造（Risk-rule-guided Reward Shaping, RRS）
- **核心思想**：在原始投资组合收益的基础上，加入基于投资组合风险规则的辅助奖励，使训练奖励更关注长期风险-收益权衡。
- **具体形式**：  
  \(R_t = r_t^{\text{port}} - \lambda_m DD_t^2 + r_{LLM}^{int,t} + \sum_{j \in S_{R_m}} \alpha_{m,j} \rho_j(\tau_t)\)  
  其中\(DD_t\)为当前回撤，\(r_{LLM}^{int,t}\)为LLM生成的内在奖励，\(\rho_j\)为选定的行为风险规则（如集中度惩罚、换手率惩罚、回撤控制等）。
- **LLM参与**：从规则库中选择2-4条风险规则并设定系数，同时生成内在奖励函数（must use enhanced state）。

#### 2.3 诊断引导的精炼（Diagnostic-guided Refinement, DGR）
- **核心思想**：通过PPO在历史数据上的滚动测试，计算候选接口的诊断指标（因子信息系数、奖励稳定性、投资组合行为等），并将这些诊断反馈给LLM，以便在下一轮迭代中有针对性地修改（保留有效因子、移除不稳定奖励项、调整风险规则等）。
- **流程**：经过M轮精炼后，根据评估窗口历史数据上的表现选出最佳接口，并在测试时完全冻结。

#### 2.4 离线设计与评估分离
- 接口设计仅使用测试窗口之前的离线数据，设计完成后冻结，测试时不更新、不查询LLM。这确保了方法严格遵循时间序贯，无前瞻偏差。

## 摘要
我们介绍了 Nemotron 3 Ultra，一个总参数 5500 亿、活跃参数 550 亿的混合专家混合 Mamba-Attention 语言模型。我们在 20 万亿文本 token 上预训练了 Nemotron 3 Ultra，随后将上下文长度扩展到 100 万 token，并使用监督微调 (SFT)、强化学习 (RL) 和多教师在线策略蒸馏 (MOPD) 进行后训练。Nemotron 3 Ultra 是我们目前性能最强的模型，采用了多项关键技术——LatentMoE、多 token 预测 (MTP)、NVFP4 预训练、多环境 RLVR、MOPD 以及推理预算控制。与最先进的公开可用大型语言模型相比，Nemotron 3 Ultra 的推理吞吐量最高可提升约 6 倍，同时保持同等精度。顶尖的精度、高推理吞吐量和 100 万 token 的上下文长度使 Nemotron 3 Ultra 成为长时间运行的自主代理任务的理想选择。我们在 HuggingFace 上开源了基础、后训练及量化检查点，以及训练数据和方案。

## Abstract
We introduce Nemotron 3 Ultra, a 550 billion total and 55 billion active parameter Mixture-of-Experts Hybrid Mamba-Attention language model. We pre-trained Nemotron 3 Ultra on 20 trillion text tokens, then extended the context length to 1M tokens, and post-trained using Supervised Fine Tuning (SFT), Reinforcement Learning (RL), and Multi-teacher On-Policy Distillation (MOPD). Nemotron 3 Ultra is our most capable model yet, employing multiple key technologies - LatentMoE, Multi Token Prediction (MTP), NVFP4 pre-training, multi-environment RLVR, MOPD, and reasoning budget control. Nemotron 3 Ultra achieves up to ~6x higher inference throughput as compared to state-of-the-art publicly available LLMs while attaining on-par accuracy. The state-of-the-art accuracy, high inference throughput, and 1M token context length make Nemotron 3 Ultra ideal for long-running autonomous agentic tasks. We open-source the base, post-trained, and quantized checkpoints, along with the training data and recipe on HuggingFace.

*   **后训练技术流程: SFT -> RLVR -> MOPD**
    *   **监督微调 (SFT)**：构建了庞大且多样化的SFT数据集，覆盖代码、数学、科学、安全、搜索、终端使用、软件工程等多个领域，初步建立模型的基础和代理能力。
    *   **统一强化学习 (RLVR)**: 在一个结合了多种代理、推理、代码、安全等环境的混合训练中使用可验证奖励的强化学习，全面提升模型各方面能力。
    *   **多教师在线策略蒸馏 (MOPD)**: 这是后训练的核心创新。
        *   **训练专业教师**：首先针对特定领域（如软件工程、搜索、办公任务）训练了超过10个专精化的教师模型。
        *   **在线策略蒸馏**：然后，学生模型基于提示生成轨迹（在线策略），并接收来自相应领域教师模型的密集token级奖励信号进行学习，将多个教师的专长合并到一个模型中。这个过程可以异步进行，高效利用计算资源。
        *   **MOPD预热**：在MOPD前增加一个轻量SFT预热阶段，让学生模型的输出分布先与教师模型的分布对齐，显著提升了代理领域的蒸馏效果。
    *   **MTP Boosting**: 针对训练和推理时MTP输入分布的不一致性，通过训练让MTP头匹配推理时在噪声输入下主干的输出分布，从而延长了投机解码的接受长度。

*   **推理优化**
    *   **NVFP4后训练量化 (PTQ)**：采用多种策略（混合精度、4/6权重缩放、FP8选择性量化）将模型量化爲5.03比特/元素的低精度格式，以在精度损失最小的情况下最大化推理效率。
    *   **Mamba SSM缓存优化**：对Mamba状态缓存进行量化和周期性检查点（checkpointing）优化，减少了内存占用和读取压力。
    *   **推理服务优化**：采用预填充-解码分离、拓扑感知的NVLink域放置和NUMA绑定等技术，以提升大规模推理服务的稳定性和吞吐量。

### 3. 实验设计

*   **数据集/场景与基准 (Benchmarks)**
    实验覆盖了广泛的评估基准，旨在全面测试模型的代理能力、推理能力、知识和效率。
    *   **代理能力**: TerminalBench 2.1, GDPVal, SWE-Bench (Verified & Multilingual), ProfBench (Search), PinchBench, TauBench V3, BrowseComp, Vals.ai Financial Agent。
    *   **推理与知识**: IOI 2025, LiveCodeBench v6, IMOAnswerBench, MathArena Apex-Shortlist, GPQA, MMLU-Pro, HLE, SciCode, CritPt, AA-Omniscience。
    *   **对话与指令遵循**: IFBench, Multi-Challenge。
    *   **长上下文**: AA-LCR, RULER (up to 1M), LongBench v2。
    *   **多语言**: MMLU-ProX, WMT24++。
    *   **推理效率**: 在8K输入/64K输出等多种配置下测量了输出token/秒/GPU的相对吞吐量。

*   **对比方法 (Baselines)**
    主要与顶尖的开源模型进行对比，包括：
    *   GLM-5.1-754B-A40B
    *   Kimi-K2.6-1T-A32B
    *   Qwen-3.5-397B-17B
    *   DeepSeek-V3.2 / V4-Pro / V4-Flash
    *   MiniMax-2.7
    *   Mistral Large 3

### 4. 资源与算力

*   **硬件**: 预训练和推理主要在NVIDIA GB200 NVL72节点上进行，利用了其NVLink高速互联。量化实验对比了B300和H100。RL训练也运行在包含GB200的集群上，并使用了NVIDIA Grace CPU进行NUMA绑定优化。
*   **并行策略**: 预训练中使用了32路上下文并行、8路张量并行、128路专家并行和2路流水线并行。推理服务时，则根据吞吐优先或延迟优先分别选择宽专家并行或张量并行。
*   **预训练数据量**: 20万亿（20T）文本token。
*   **训练稳定性与问题**: 预训练过程中遇到了两次训练发散（loss spike）。第一次归因于MTP模块的输出层梯度累加精度过低（从FP32降至BF16），通过回滚检查点并恢复FP32修复。第二次原因未定，但发现提前开始学习率退火可以缓解，最终将总训练token量从原计划调整至20T。
*   文中未给出确切的总训练时长或总GPU数量，但细节描述表明其使用了一个大规模的计算集群。

### 5. 实验数量与充分性

*   **实验数量**: 报告了大量的实验。
    *   **消融研究**: 包括数据混合（法律、事实类数据影响）、模型稳定性（BF16回滚）、NVFP4训练（精度损失<0.4%）、MOPD预热、MTP Draft长度扫描、量化精度（BPE选择）、FP4算法选择、Mamba缓存精度等。
    *   **多维度评估**: 后训练模型在30多个不同类型的基准测试上进行了系统评估，并与6个以上的主流前沿模型进行了对比。
    *   **推理测试**: 全面测试了在不同输入输出长度、batch size、精度下的推理吞吐量。
*   **实验充分性与客观性**:
    *   *充分性*: 实验设计非常详尽，从数据配比、架构组件、训练稳定性到最终的量化部署都进行了打磨和验证。特别是PTQ阶段的BPE扫描和算法对比，清晰地展示了选择最优量化配置的依据。
    *   *客观公平性*: 模型主要与最强的开源模型对比。所有模型的评估设置（如提示词模板、工具配置）保持一致，并使用了统一的评估框架。使用了PinchBench和ProfBench作为“保留验证门”（held-out generalization gates），仅在最终模型上测试一次，有效防止了针对特定基准的过拟合，增强了结果的可信度。

### 6. 论文的主要结论与发现

*   **性能与效率的突出平衡**：Nemotron 3 Ultra 550B-A55B 在多种代理和推理基准测试上，达到了与总参数量高出很多的顶尖模型（如GLM-5.1 744B, Kimi-K2.6 1T）相当的精度，同时实现了最高**5.9倍**的推理吞吐量提升。
*   **后训练新范式成效显著**：提出的“通用SFT -> 统一RLVR -> 多教师MOPD”后训练流程能有效整合多个专精教师的能力，在代理任务（如Terminal Bench, GDPVal）上成功恢复了大部分教师相对于学生的性能差距，某些情况下甚至超越了单领域教师。
*   **关键技术组合有效**：混合Mamba-Attention架构、LatentMoE、MTP、NVFP4量化等一系列技术共同作用，使得模型在效率、精度和长上下文支持上具备全面优势。
*   **大规模NVFP4预训练的可行性**：验证了FP4精度进行稳定预训练的可行性，其训练损失与高精度基准的相对差距低于0.4%，为极低精度训练树立了标杆。
*   **推理效率分析深刻**：揭示了模型在不同工作负载下的瓶颈：预填充阶段受活跃参数计算量制约，而大批量解码阶段受总参数权重的I/O制约，Mamba架构在解码阶段优势明显。

### 7. 优点

*   **架构系统性创新**：不是单一技术的应用，而是端到端地联合优化模型架构、训练算法、数据配方和推理堆栈，实现了“1+1 > 2”的效果。
*   **极强的工程实践性**：论文不仅提出了算法，还详细记录了从预训练发散修复、多教师在线蒸馏的异步管道设计，到大规模RL基础设施的优化（启动开销降低99%、吞吐量提升20%）等大量工程细节，对社区有很高的参考价值。
*   **极致效率导向**：在模型设计的每个环节都考虑了推理效率，如其混合架构、MTP头设计、NVFP4全流程训练等，使得最终产品在实用部署上具有很强的竞争力。
*   **开源全面性**：一次性开源了基础模型、后训练模型、量化模型、训练数据（多个新数据集）及整个训练方案，展现了全面的开放性。

### 8. 不足与局限

*   **MOPD方法的限制**：MOPD在那些需要学生模型本不具备的全新推理能力（主要由教师通过大量额外SFT获得）的任务上效果有限，因为这些教师知道的路径学生自己极少采样，导致监督信号弱。文中尝试的Logit匹配等方法表现不佳。
*   **长时任务训练效率**：混合端到端代理环境与单轮推理环境进行MOPD训练时，因轨迹生成时间差异巨大，导致训练效率低下。当前方案主要通过单轮PivotRL进行近似，但端到端训练的潜在收益仍有待探索。
*   **训练稳定性问题未完全根除**：第二次预训练发散的原因未被彻底查明，仅通过提前退火学习率规避，并观察到专家负载不平衡、残差流范数异常等现象，但未给出根本解决方案。
*   **极端长上下文能力下限**：虽然支持1M上下文，但在某些长上下文评测（如LongBench v2）中并非最佳，且在小批量和预填充阶段，其吞吐量优势会减弱。
*   **模型规模与部署门槛**：550B的总参数量对显存有极高要求，虽然量化后有所缓解，但依然远超普通消费级和研究级硬件的承载能力，限制了低资源环境的探索和复现。
*   **泛化性测试**：虽然使用了两个保留验证集，但所有评估均基于现有基准。模型在真实世界、开放域、长时间运行的代理任务中的表现和鲁棒性仍需更广泛的社区检验。

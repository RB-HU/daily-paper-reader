## 摘要
基于大语言模型的智能体日益频繁地处理具有相互依赖决策的长程任务，每个动作都会重塑未来的约束，中间错误可能级联放大。现有的检索增强生成和智能体记忆系统按语义相似性组织历史，在决策时刻检索内容相关的条目。我们认为这种设计与执行状态依赖不匹配：它割裂了决策轨迹，混合了有效和错误的痕迹，阻碍了连贯的状态重建和错误隔离。我们提出 MAGE（记忆作为智能体引导的探索），一种主动的执行状态管理器，将交互存储在一个层级状态树中。智能体从活跃的根到当前路径推导其状态，结合子目标摘要、近期痕迹以及先前分支的提示。四个耦合操作维护该树：Grow 记录新痕迹，Compress 摘要已完成的子目标，Maintain 验证摘要，Revise 恢复目标边界并在新分支上继续。这种设计在保持状态完整性和将错误片段从活跃路径中隔离的同时，限制了上下文的增长。在 MemoryArena 上的实验表明，MAGE 将平均任务成功率较基线提高了 7.8 至 20.4 个百分点，同时减少了 55.1% 的令牌消耗。

## Abstract
LLM-based agents increasingly tackle long-horizon tasks with interdependent decisions, where each action reshapes future constraints and intermediate errors can cascade. Existing RAG and agent memory systems organize histories by semantic similarity, retrieving content-relevant entries at decision time. We argue that this design mismatches execution-state dependencies: it fragments decision trajectories and mixes valid and erroneous traces, hindering coherent state reconstruction and error isolation. We propose MAGE (Memory as Agent-Guided Exploration), an active execution-state manager that stores interactions in a hierarchical state tree. The agent derives its state from the active root-to-current path, combining subgoal summaries, recent traces, and hints from prior branches. Four coupled operations maintain the tree: Grow records new traces, Compress summarizes completed subgoals, Maintain validates summaries, and Revise restores a target boundary and resumes on a new branch. This design bounds context growth while preserving state integrity and isolating flawed segments from the active path. Experiments on MemoryArena show that MAGE improves the average task success rate by 7.8--20.4 pp over baselines, while reducing token consumption by 55.1%.

    1.  **执行状态碎片化**：决策轨迹被割裂成孤立的语义片段，破坏了执行状态的连贯性和完整性，智能体可能基于不完整的状态做出错误决策。

*   **整体含义**：论文主张将长程智能体的记忆设计，从“**语义驱动的档案库**”根本性地转向“**执行状态管理器**”，通过主动管理和维护决策过程中的状态结构，而非被动检索，来应对上述挑战。

### 2. 论文提出的方法论

*   **核心思想**：MAGE（Memory as Agent-Guided Exploration）将智能体的历史交互组织为一棵持久的**两层级状态树**，智能体直接从树的活跃路径中读取执行状态，而非通过语义检索拼凑。
*   **关键技术细节**：
    *   **层级状态树**：
        *   **底层**：记录原始的动作-观测对，保留细粒度状态依赖。
        *   **顶层**：存储由底层片段压缩而成的**子目标摘要节点**，在子目标边界处实现有界压缩。
    *   **执行状态表示**：智能体当前的执行状态`S`由三部分组成：
        1.  **压缩状态 `C`**：从根到当前顶层节点路径上的所有子目标摘要。
        2.  **原始状态 `R`**：自上次压缩后，最近累积的底层原始记录。
        3.  **执行提示 `H`**：来自当前节点子节点（先前探索分支）的信息及诊断反馈。
    *   **闭环状态管理操作（算法流程）**：
        1.  **Grow**：在底层追加新的动作-观测节点。若内容与先前分支相同，则自动合并，避免重复。
        2.  **Compress**：在子目标完成或`R`超出长度阈值时，将该子目标的底层片段压缩成一个顶层摘要节点，并清空`R`，限制上下文增长。
        3.  **Maintain**：紧接压缩后，对新的摘要节点进行**边界级验证**。LLM会检查摘要信息完整性、任务要求满足度等。验证失败，则记录诊断反馈并触发修正。
        4.  **Revise**：当`Maintain`失败或智能体主动检测到错误时，将`pt`（顶层指针）和`pb`（底层指针）回滚至目标边界，注入诊断反馈至`H`，并从该点开始一条**新的执行分支**。这保留了有效历史，并结构性地隔离了错误片段。

### 6. 论文的主要结论与发现

1.  **语义记忆范式的失效**：在具有复杂状态依赖的任务（如网页购物、网页搜索）上，RAG和记忆系统基线的任务成功率显著低于长上下文方法，证实了语义检索导致的**状态碎片化**会破坏执行完整性。而在依赖稀疏、实体明确的任务（如旅行规划、形式化推理）上，基于语义的方法则表现尚可。

4.  **机制验证**：消融实验表明，`Compress`（边界压缩）、`Maintain`（状态验证）和`Revise`（错误隔离与分支）三个机制对于MAGE的可靠状态管理都至关重要，任一缺失都会导致性能下降。

### 7. 优点

*   **范式创新**：从根本上改变了智能体记忆的构建哲学，从被动检索转向主动管理执行状态。
*   **设计巧妙**：两层级树结构自然地解决了上下文长度与状态完整性的矛盾；通过**分支**而非覆盖来实现修正，优雅地实现了**错误隔离**，保留了过去探索的收益。
*   **机制互补**：四个闭环操作（Grow, Compress, Maintain, Revise）环环相扣，形成了一个完整的认知-执行管理循环，逻辑自洽。
*   **效果显著**：在多个领域和不同后端模型上，同时实现了性能（成功率）和效率（令牌消耗）的最优，达到了论文开篇图1所示的理想区间。

### 8. 不足与局限

*   **LLM决策依赖性**：`Compress`和`Maintain`操作依赖于LLM在子目标边界做出正确判断，如果摘要本身有误，`Maintain`可能检测不到，导致错误记忆成为可信任状态。
*   **通用性与可扩展性**：虽然在四个领域上得到了验证，但MemoryArena本身可能还是实验室环境。在更加开放、动态、多模态的真实世界场景中，子目标边界的自动识别和状态摘要的生成将面临更大挑战。
*   **计算开销**：虽然相比于长上下文节省了卷积，但MAGE的`Compress`和`Maintain`操作需要**额外的LLM调用**。在那些观察数据短、子目标切换极频繁的高交互任务中，这部分开销的优势可能不再明显。
*   **实验覆盖**：虽然做了后端模型泛化实验，但所有实验都是在单一基准上进行的，未在同类型的其他长程任务基准上验证MAGE的鲁棒性。

## 摘要
通过从专家轨迹的模仿学习训练交互式网络代理已成为一种高效方法。然而，确定专家干预的最佳时机是该领域的关键挑战。延迟干预往往导致早期错误累积，使页面状态陷入不可恢复的境地；而过早或过度干预则会使代理过度依赖专家策略，陷入单一刚性轨迹的局部最优。我们提出了推测性回滚校正（SRC），一种用于可重置代理环境的分支级模仿框架。该方法不要求在每个访问状态请求教师标签，也不仅在完成整个轨迹后才进行校正，而是采用固定视界分支评审：学生执行一小段推测性动作序列，然后由教师进行评审，教师仅在局部进展中断时定位首个有害偏差。回滚机制保留了有用的前缀，而成功的执行路径则由硬验证器过滤，并保留在一个轻量级质量多样性存档中。由此产生的数据支持对局部校正和通过验证器的轨迹进行下一动作监督微调。在WebArena-Infinity环境中，SRC收集了977条通过验证器的轨迹和9,183个下一动作示例；固定视界评审相比于逐步评审改进了恢复与查询的权衡，同时保留了通过验证器的解决方案变体。代码可在https://github.com/LongkunHao/SRC_gui_agent获取。

## Abstract
Training interactive web agents through imitation learning from expert trajectories has emerged as a highly effective approach. However, determining the optimal timing for expert intervention presents a critical challenge in this context. Delayed intervention often leads to the accumulation of early-stage errors, pushing the page state into an irrecoverable regime. Conversely, premature or excessive intervention causes the agent to become overly reliant on expert policies, trapping the model in local optima characterized by a single, rigid trajectory. We propose Speculative Rollback Correction (SRC), a branch-level imitation framework for resettable agent environments. Instead of requesting teacher labels at every visited state or correcting only after a completed trajectory, SRC uses fixed-horizon branch review: the student executes a short speculative segment before teacher review, and the teacher localizes the first harmful deviation only when local progress breaks. Rollback preserves useful prefixes, while successful rollouts are filtered by a hard verifier and retained in a lightweight quality-diversity archive. The resulting data supports next-action supervised fine-tuning on both localized corrections and verifier-passing trajectories. On WebArena-Infinity, SRC collects 977 verifier-passing trajectories and 9,183 next-action examples; fixed-horizon review improves the recovery-versus-query tradeoff over step-level review while retaining verifier-passing solution variants. Code is available at https://github.com/LongkunHao/SRC_gui_agent.

## 2. 方法论

### 2.1 核心思想
- 将教师监管的粒度从**单步**改为**短推测分支**。
- 学生每次执行一个固定长度 K 的动作序列（分支），教师**仅当分支造成局部进展中断时**才介入，并精确定位分支中第一个有害动作。
- **回滚**保留该有害动作之前的有效前缀，重新放出环境状态，由教师提供一个校正动作，随后学生继续自主执行。
- 成功完成的轨迹由硬验证器判定，通过**质量多样性存档**保留不同行为模式的高质量解，避免只保留最短或最“专家式”的路径。

### 2.2 算法流程（文字描述）
SRC 的数据收集过程如 Algorithm 1 所示：
1. 初始化环境和空轨迹 τ、空校正集 Dcorr。
2. 在未终止且预算未用尽时：
   - 让学生策略 πθ 执行一段至多 K 步的推测分支 Bt。
   - 让教师评审器 Tr 对该分支给出 **ACCEPT** 或最早有害步骤的索引 j。
   - 若接受，将分支全部动作加入 τ。
   - 若拒绝，保留分支前 j-1 步，回滚环境至对应状态，调用教师校正器 Tc 生成校正动作 a*ⱼ，执行该动作并将 (x, hⱼ, a*ⱼ) 加入 Dcorr，然后从校正后状态继续学生策略。
3. 返回完整轨迹 τ、校正集 Dcorr 及硬验证器结果。

### 2.3 训练数据与目标
- 训练集由两部分组成：
  - 局部回滚校正样本 Dcorr（状态为学生引发的分布，动作为教师校正动作）。
  - 从质量多样性存档中提取的完整成功轨迹的下一动作标签 Darc。
- 仅使用下一动作监督微调（SFT）：
  \[
  \mathcal{L}(\theta) = \mathbb{E}_{(x,h,a) \sim D_\text{sft}} [-\log \pi_\theta(a \mid h, x)]
  \]
  其中 \(D_\text{sft} = D_\text{corr} \cup D_\text{act arc}\)，不引入奖励模型或偏好优化。

### 2.4 质量多样性存档
- 每个候选轨迹 τ 具有质量向量 \(q(\tau) = (|\tau|, \text{repeat}(\tau), \text{interventions}(\tau))\) 和行为描述符 b(τ)（如路径长度桶、主导行为类型、教师干预次数桶）。
- 存档仅保留通过验证器且满足质量约束（长度 ≤ 60、重复动作 ≤ 4、干预 ≤ 6）的成功轨迹，每个描述符桶保留少量高品质轨迹。
- 这使得训练数据覆盖多种解法模式，而非仅保留最短或最少干预的路径。

## 6. 主要结论与发现

1. **显著提升成功率**：Expert SFT + SRC 在 WebArena-Infinity 上达到 35.0% SR，比 Expert SFT（25.3%）提升 9.7 个点；在 OSWorld 子集上提升幅度更大（+12.9）。
2. **优于其它在线校正方法**：OEC 因随机切换导致性能低于 Expert SFT；LEAP 虽然提升 SFT，但每步需教师查询，成本高而最终效果不及 SRC。
3. **固定视界分支评审可改善恢复-查询权衡**：K=3 在成功率和查询次数间取得最佳平衡，比步级评审（K=1）查询少、成功率高；K=7 查询更少但成功率开始下降。
4. **数据构成健康**：仅有 14.2% 的下一动作示例来自教师干预，大部分为学生自主分支，避免了教师轨迹主导。
5. **保留解法多样性**：质量多样性存档的 bins 随收集轮次增加而扩展，SRC 能覆盖更多成功状态簇，而不像 OEC 那样坍缩至少数专家路径。

## 7. 优点

- **方法创新**：首次将 DAgger 式在线专家校正系统地适配到视觉 GUI 代理，并引入推测性分支评审、精细回滚和质量多样性存档，设计巧妙且系统。
- **解耦三阶段判断**：明确分离局部进展（教师评审）、最终成功（硬验证器）和训练价值（QD 存档），解决了干预时机和多样性之间的矛盾。
- **实验全面**：涵盖多个基准和迁移场景，消融实验揭示视界 K 的权衡，协变量偏移分析验证了减少有害漂移与保持多样性的双重效果。
- **低成本部署**：最终模型完全不依赖教师，测试时无额外查询开销。
- **框架柔性强**：模型与模态无关，可作为通用训练范式推广到其他长视界交互任务。

## 8. 不足与局限

- **环境重置假设**：SRC 依赖可重置、可回滚的环境，对不可逆的网页工作流、隐藏状态变化或真实在线系统不适用。
- **固定视界限制**：当前使用固定 K，未根据任务结构自适应（如子目标分解、动态触发评审），文中将此列为未来工作。
- **安卓等平台未评估**：实验局限于桌面浏览器和部分桌面应用，未在移动 GUI 环境测试。
- **缺乏训练成本细节**：论文未报告 SFT 训练所需的 GPU 资源、时间或能耗，难以评估实际训练开销。
- **视觉多样性度量简单**：协变量偏移分析基于冻结的 VLM 嵌入，可能无法完美捕捉行为差异。

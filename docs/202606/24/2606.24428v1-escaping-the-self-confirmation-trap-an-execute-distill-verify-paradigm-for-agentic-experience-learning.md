## 摘要
经验驱动的自我进化对于大型语言模型（LLM）智能体通过开放世界交互实现改进至关重要。然而，现有的经验学习方法大多依赖单智能体循环：同一智能体执行任务、总结结果并决定记忆内容。这种设置使智能体容易陷入“自我确认陷阱”：错误但自洽的轨迹被误认为成功经验，导致在检索和重用过程中累积错误。为了解决这一问题，我们提出了EDV，一种用于可靠经验学习的执行-提炼-验证框架。在执行阶段，多个异构智能体并行探索相同的任务空间，生成多样化的候选轨迹。在提炼阶段，一个专用的第三方智能体比较分析这些轨迹，产生候选经验，减少以执行者为中心的总结偏差。在验证阶段，执行团队通过共识机制验证候选经验，只有批准的经验才会被写入共享或私有记忆。通过解耦这三个阶段，EDV将经验学习从孤立的自我反思转变为协作构建，在记忆插入之前过滤错误和噪声内容。我们在三个具有挑战性的长时间基准上评估了EDV：tau2-bench、Mind2Web和MMTB。结果表明，EDV始终优于强基线，验证了可靠的经验构建对于鲁棒的智能体自我进化至关重要。我们的代码可在https://github.com/shidingz/EDV获取。

## Abstract
Experience-driven self-evolution is critical for large language model (LLM) agents to improve through open-world interaction. However, existing experience learning methods mostly rely on single-agent loops, where the same agent executes tasks, summarizes outcomes, and determines memory content. This setup makes agents vulnerable to the Self-Confirmation Trap: wrong-but-self-consistent trajectories are misidentified as successful experience, leading to cumulative errors during retrieval and reuse. To address this issue, we propose EDV, an Execute-Distill-Verify framework for reliable experience learning. In the Execute stage, multiple heterogeneous agents explore the same task space in parallel to generate diverse candidate trajectories. In the Distill stage, a dedicated third-party agent comparatively analyzes these trajectories to produce candidate experiences, reducing executor-centric summarization bias. In the Verify stage, the execution group validates candidates via a consensus mechanism, and only approved experiences are written into shared or private memory. By decoupling the three stages, EDV transforms experience learning from isolated self-reflection into collaborative construction, filtering erroneous and noisy content before memory insertion. We evaluate EDV on three challenging long-horizon benchmarks: tau2-bench, Mind2Web and MMTB. Results show EDV consistently outperforms strong baselines, validating that reliable experience construction is essential for robust agent self-evolution. Our code is available at https://github.com/shidingz/EDV.

## 2. 方法论：EDV 框架

- **总体设计**：EDV 框架包含两个层级：
  - **经验构建阶段**（离线）：通过异构执行、第三方提炼和共识验证产生高质量经验，并写入共享/私有记忆库。
  - **推理使用阶段**（在线）：根据能力矩阵选择合适的求解模型，分层检索共享和私有记忆，辅助决策。
- **Execute（执行）**：
  - 从异构智能体池 \(A = \{A_1, A_2, \dots, A_K\}\) 中随机采样形成**执行组** \(A_{\text{exec}}\)，每个智能体独立与环境交互，生成候选轨迹集合 \(T(q)\)。
  - 目的：扩大解空间覆盖，提供多样化的成功/失败模式作为后续比较的素材。
- **Distill（提炼）**：
  - 从池中选出一个**第三方提炼智能体** \(A_{\text{distill}}\)，对多条轨迹进行**跨轨迹对比分析**，提炼候选经验集 \(E_{\text{cand}}\)。
  - 优势：将经验构建从“执行者中心的自总结”转变为中性第三方对比分析，减少单一视角偏差。
- **Verify（验证）**：
  - 由原执行组构成**验证组** \(A_{\text{verify}} = A_{\text{exec}}\)，对每条候选经验根据自身执行上下文给出二值判断 \(V_j(e)\)。
  - 采用**严格默认拒绝策略**：
    - 全票通过 → 写入**共享记忆库**；
    - 仅部分通过 → 写入相应智能体的**私有记忆库**；
    - 否则 → 丢弃。
  - 这种共识验证在记忆写入之前提高门槛，阻断低质量经验的长期污染。
- **记忆存储与推理使用**：
  - 维护**共享记忆库**和**私有记忆库**。
  - 维护**能力矩阵**：记录哪种任务类型更擅长由哪个智能体处理。
  - 新任务到来时：由能力矩阵匹配最佳求解智能体 → 分层检索（先共享，再必要时查私有）→ 将检索到的记忆附加到上下文，辅助推理。

## 6. 主要结论与发现

- **EDV 一致显著优于所有基线**：在 τ2‑bench、Mind2Web 和 MMTB 上，EDV 均取得最优或接近最优的成绩，相比于单模型记忆或简单路由方案有大幅提升。
- **可靠的经验构建比经验数量更重要**：人为注入少量错误经验即导致 ReasonBank 性能明显下降（Pass@1 从 82.5 降至 77.2），而 EDV 通过验证机制有效阻止了低质量记忆入库。
- **Execute‑Distill‑Verify 三阶段缺一不可**：单智能体自我验证无法摆脱自确认偏见；独立验证器缺乏多样化轨迹而增益有限；多智能体执行 + 第三方提炼才能充分释放轨迹多样性；共识验证进一步过滤残余错误。
- **记忆分层和能力矩阵具有实质贡献**：共享记忆在 72.3% 的任务中被检索，贡献约 2.3% 性能；私有记忆在 31.8% 的任务中被使用，贡献约 0.6%，两者互补，覆盖通用规则与个性化边缘案例。
- **EDV 产生更高质量的记忆**：从单智能体的局部操作提示升级为状态感知、战略级、可执行的经验，显著减少冗余和错误传播。

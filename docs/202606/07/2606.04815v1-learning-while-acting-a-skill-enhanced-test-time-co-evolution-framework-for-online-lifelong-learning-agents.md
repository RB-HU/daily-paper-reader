## 摘要
终身学习对于在动态、交互环境中运行的大语言模型（LLM）智能体至关重要。然而，现有的用于长时程任务的终身学习智能体在推理过程中通常依赖于离散技能或基于过去经验的检索，且参数保持静态，这阻碍了它们像人类学习者那样持续内化测试时的反馈。为弥补这一差距，我们提出了技能增强测试时协同进化（LifeSkill），这是一个面向在线终身学习智能体的两阶段强化学习框架。具体而言，我们设计了验证器引导的技能学习，通过根据多个技能条件策略展开的平均验证器成功率来奖励候选技能，从而解决技能提取缺乏直接监督的问题，鼓励模型生成对解决任务有用的技能，而不仅仅是文本上看似合理的技能。此外，我们引入了在线技能内化，通过在测试时交互过程中将技能条件轨迹转化为奖励信号，持续改进策略模型。这使智能体能够直接将推理能力内化到其参数中，避免了经验检索导致的上下文膨胀。在 LifelongAgentBench 上的实验表明，与现有的终身智能体基线相比，LifeSkill 将平均性能提高了 7 个绝对百分点。

## Abstract
Lifelong learning is essential for Large Language Model (LLM) agents operating in dynamic, interactive environments. However, existing lifelong learning agents for long-horizon tasks typically depend on discrete skill or past experiences retrieval with static parameters during inference, which prevents them from continuously internalizing test-time feedback like human learners. To bridge this gap, we propose Skill-enhanced Test-Time Co-Evolution (\texttt{LifeSkill}), a two-stage reinforcement learning framework for Online Lifelong Learning Agents. Specifically, we design Verifier-Guided Skill Learning that addresses the lack of direct supervision for skill extraction by rewarding candidate skills according to the average verifier success of multiple skill-conditioned policy rollouts, encouraging the model to generate skills that are useful for solving tasks rather than merely plausible in text. Furthermore, we introduce Online Skill Internalization, which continuously improves the policy model during test-time interaction by transforming skill-conditioned trajectories into reward signals. This enables the agent to directly internalize reasoning capabilities into its parameters, avoiding the context bloat of experience retrieval. Experiments on LifelongAgentBench show that LifeSkill improves average performance by 7 absolute points by comparing with existing lifelong agent baselines.

- **关键痛点**：这种“**静态参数+外部检索**”的模式导致智能体无法像人类一样在部署过程中持续从测试时的**失败反馈中内化知识**，使得模型参数本身无法真正成长。同时，过度依赖外部文本检索会造成**上下文窗口不断膨胀（context bloat）**，阻碍了持续适应能力。
- **整体含义**：本文旨在改变这一现状，提出一个**技能增强的测试时协同进化框架**，使LLM智能体能够在在线交互过程中“边行动边学习”，将执行反馈实时转化为**参数化能力**，实现真正的终身学习。

### 2. 论文提出的方法论：LifeSkill框架

LifeSkill是一个**两阶段强化学习框架**，包含一个**策略模型**（Policy Model，负责执行任务）和一个**技能提取模型**（Skill Extraction Model，负责从失败中提取经验），两者在任务流上协同进化。

- **核心思想**：将解决问题的行为从**外部文本技能**逐步**内化到策略模型的参数**中，形成**测试时协同进化**闭环。

- **关键技术细节**：
    - **第一阶段：验证器引导的技能学习（Verifier-Guided Skill Learning, VGSL）**
        - **目的**：训练技能提取模型生成真正有用的技能，而非文字上合理的空洞描述。
        - **流程**：
            1.  当策略模型执行任务失败后，技能提取模型根据 `（问题, 失败轨迹）` 生成多个候选技能。
            2.  策略模型再以每个候选技能为条件，进行多次尝试（rollout）。
            3.  利用任务的**确定性二元验证器**评估这些尝试的成功率，作为该候选技能的**效用分数**。
        - **优化**：采用 **DAPO 算法（DeepSeek-Prover-Verifier的变体，一种Token级别的PPO式离线策略优化）**，以候选技能组内标准化的效用分数为优势值，直接优化技能提取模型，使其学会生成高成功率的技能。

    - **第二阶段：在线技能内化（Online Skill Internalization, OSI）**
        - **目的**：将在VGSL阶段通过技能引导成功的轨迹，转化为策略模型的参数更新。
        - **核心操作**：**技能移除（Skill Removal）**。从成功的 `（问题, 技能, 成功轨迹）` 三元组中**删除掉技能文本**，仅保留原始 `（问题, 成功轨迹）` 作为训练数据。
        - **优化**：让策略模型在仅有原始任务输入的条件下，最大化生成那条成功轨迹的对数似然（Log-Likelihood），相当于**奖励加权的策略优化**。这样，策略模型学会了在没有外部技能提示的情况下，也能复现成功行为，实现了能力内化。

    - **协同流程**：VGSL利用策略模型的rollout结果来优化技能提取模型；OSI则利用提取出的优质技能来优化策略模型。两者相互促进，形成“更好的技能→更优的探索→更强的策略→更好的技能提取”的**在线协同进化循环**。

- **协同进化是关键**：消融实验证实，VGSL和OSI两个阶段缺一不可，它们的协同作用是实现持续性能增长的核心驱动力。
- **技能质量可进化**：验证器引导的训练使模型生成的技能从早期的通用、模糊描述，逐步演化为精准、可执行的操作指南，从而能有效纠正策略错误。

### 7. 优点

- **范式创新**：将终生学习从“外部记忆检索”和“测试时搜索”范式中解放出来，提出了“**测试时参数进化**”的新思路，实现了真正的在线学习。
- **机制精巧**：通过**技能提取与内化**的巧妙解耦和协同闭环，系统性地解决了如何从失败中学习、如何将文本经验固化为模型能力的双重难题。
- **执行反馈驱动**：使用确定性的环境验证器作为奖励信号来训练技能，避免了LLM文本判别器（Judge）的主观性和不精确性，使学习目标与任务成功直接对齐。
- **避免上下文膨胀**：OSI机制将技能内化，使得部署时不再需要冗长的技能提示，有效控制了上下文长度，为长周期任务的持续运行提供了可能。

### 8. 不足与局限

- **稀疏奖励场景性能不足**：在知识图谱（KG）任务上，LifeSkill的性能（0.32）未能超越最强的免训练基线（0.36）。论文分析认为，这可能是由于KG任务轨迹更长，导致二元奖励信号过于稀疏，削弱了VGSL和OSI的学习动力。
- **测试时计算成本高**：VGSL阶段需要对每个失败任务采样多个候选技能，并对每个技能进行多次策略rollout，这会带来相当大的测试时计算开销，可能影响实时性要求高的应用。
- **对验证器的依赖**：方法有效性高度依赖于**准确、无噪声的二元环境验证器**。在验证信号模糊、带有噪声或非二元奖励的开放域任务中，该框架的适用性存疑。
- **缺乏遗忘性分析**：虽然声称是终身学习，但实验主要展示了“前向迁移”（在新任务上表现更好），并未深入衡量模型在学习新技能后对过往已掌握技能的遗忘程度，缺乏关于灾难性遗忘的定量分析。

## 摘要
近年来，AI智能体能够灵活调用技能解决复杂任务，但其长期改进受到缺乏系统性技能构建、积累与转移的根本制约。特别是，由于没有统一的技能整合框架，智能体倾向于在不同任务中重复构建相似能力，无法有效将经验转化为可复用的资产，也难以将任务特定的技能泛化至新场景。为解决这一局限，我们提出 SkillPyramid，一种利用既有技能经验以实现更广泛任务泛化的技能整合框架。基于分层技能拓扑结构，SkillPyramid 进一步引入自进化机制，使智能体能够在任务执行过程中组合、验证和融合新技能。在 ALFWorld、WebShop 和 ScienceWorld 上针对四种骨干模型的实验表明，SkillPyramid 将平均奖励显著提高了38.0%，并将执行步数减少了27.7%。总体而言，我们的方法将技能集合从静态资源池转变为动态演进系统。

## Abstract
Recent AI agents can flexibly invoke skills to solve complex tasks, but their long-term improvement is fundamentally constrained by a lack of systematic skill construction, accumulation, and transfer. In particular, without a unified framework for skill consolidation, agents tend to redundantly construct similar capabilities across different tasks, are unable to effectively transform experience into reusable assets, and struggle to generalize task-specific skills to novel scenarios. To address this limitation, we propose SkillPyramid, a skill consolidation framework that reuses existing skill experience for broader task generalization. Operating on a hierarchical skill topology, SkillPyramid further introduces a self-evolution mechanism that enables agents to compose, validate, and incorporate new skills during task execution. Experiments on ALFWorld, WebShop, and ScienceWorld across four backbone models show that SkillPyramid substantially increases the average reward by 38.0% and reduces execution steps by 27.7%. Overall, our method transforms a skill collection from a static resource pool into a dynamic evolution system.

### 2. 方法论：SkillPyramid 框架  
- **核心思想**：构建一个**多层级的技能金字塔**，底层为细粒度的原子技能，上层为抽象的问题解决模式。通过**向下原子提取（Downward Atomic Extraction）**和**向上抽象归纳（Upward Abstract Induction）**，将扁平技能集重新组织为有层次、可组合的结构。  
- **关键技术组件**：  
  - **关系分析器（Relation Analyzer）**：  
    - 先通过**粗糙分组**（Coarse Grouping）按技能名和描述聚类相关技能；  
    - 再对每个组进行**细粒度分析**，输出关系构建任务（技能子集、关系类型、构造依据）。  
  - **关系构建器（Relation Builder）**：  
    - **向下原子提取**：从多个技能中抽取共享的最小可复用能力，生成**原子技能**\(a_k\)，并改写原始技能使其通过“复用引用”（\(ρ(a_k)\)）显式调用该原子技能。  
    - **向上抽象归纳**：对共享高层求解模式的技能组，归纳出**抽象技能**\(u_k\)，描述技能组合逻辑而非具体执行步骤。  
    - 层位置确定：\(ℓ(a_k) = \min_{s∈g_k} ℓ(s) - 1\)，\(ℓ(u_k) = \max_{s∈g_k} ℓ(s) + 1\)。  
  - **技能创建器（Skill Creator）**：  
    - 对给定任务\(x\)，从金字塔中检索相关技能\(R(x)\)；  
    - 分两阶段生成新技能：**框架构建**（优先使用高层抽象技能规划结构），**细节实例化**（优先使用底层原子技能填充可执行操作）；  
    - 最终产出任务专有技能\(\hat{s} = C_{\text{create}}(x, R(x), P)\)。  
  - **自进化机制（Self-Evolution）**：  
    - 新创建的技能\(\hat{s}\)不会被孤立追加，而是作为查询技能，经由关系分析器与构建器**增量式**整合回金字塔\(P_{t+1} = \text{Update}(P_t, \hat{s}, ΔP_t)\)，保留并更新层次与复用关系。  
- **流程总览**：初始构建→ 任务驱动的技能创建 → 自进化扩展，形成闭环的动态演进系统。

- **层级化可复用架构**：将技能明确分为原子层和抽象层，既共享执行细节也共享问题模式，避免扁平库的冗余。  
- **任务驱动的自进化闭环**：新技能创建后即时关联到已有金字塔，避免了重新全量分析的高昂成本。

## 摘要
基于LLM的智能体能够通过结合推理与环境反馈来解决多步交互任务，但每次任务都从相同的固定语境开始，过程中发现的有用策略在任务结束后便丢失了。现有方法要么将学习局限于当前任务，要么将所有经验汇集到一个无类型区分的单一存储中，而不区分知识类型、通过使用跟踪质量，或平衡库中仍缺失的部分。我们提出统一语境演化（UCE），一种无梯度框架，将智能体经验外化为一个由类型化的可演化语境单元（ECU）组成的演化库。UCE将经验分解为四种互补类型（记忆、策略、工作流和技能），每种类型从特定条件下的轨迹生成，在决策时检索，通过重复使用结果评分，并在失去价值时剪枝。调度模块将每个周期的生成预算分配给库中最薄弱的类型。在两个交互基准测试中，UCE将ALFWorld的成功率从75.4%提升至96.3%，将WebShop任务得分从45.1%提升至61.3%，并且积累的库无需重新训练即可迁移到替代行动者骨干网络。

## Abstract
LLM-based agents can solve multi-step interactive tasks by combining reasoning with environment feedback, yet each episode starts from the same fixed context and any useful strategy discovered along the way is lost once the task ends. Existing approaches either limit learning to the current task or pool all experience into a single untyped store, without distinguishing knowledge types, tracking quality through use, or balancing what the library still lacks. We introduce Unified Context Evolution (UCE), a gradient-free framework that externalizes agent experience into an evolving library of typed Evolvable Context Units (ECUs). UCE decomposes experience into four complementary types (Memory, Strategy, Workflow, and Skill), each generated from trajectories under type-specific conditions, retrieved at decision time, scored through repeated usage outcomes, and pruned when no longer valuable. A scheduling module allocates each cycle's generation budget toward the types where the library is weakest. Across two interactive benchmarks, UCE raises ALFWorld success from 75.4% to 96.3% and WebShop task score from 45.1% to 61.3%, and the accumulated library transfers to alternative actor backbones without retraining.

### 2. 方法论：核心思想、关键技术细节与算法流程

*   **核心思想**：**统一语境演化** 框架，将智能体的成功与失败经验显式地“外部化”为一个类型化的、可进化的语境库，即**可演化语境单元**库。该库随任务迭代不断生成、评估、更新和剪枝，为后续任务提供即时的、高质量的辅助知识。
*   **关键技术细节**：
    *   **可演化语境单元**：
        *   **四种互补类型**：
            1.  **记忆**：关于环境机制的事实性知识（如“水槽可以清洁可携带物品，无需先打开水龙头”），从单条轨迹中生成。
            2.  **策略**：在特定状态下的条件性决策规则（如“如果放入失败，则先打开容器再重试”），从同类型任务的多个轨迹中生成。
            3.  **工作流**：完成任务的标准操作步骤序列（如“找到目标物 -> 改造目标物 -> 放置到目标容器”），从同类型的成功轨迹中生成。
            4.  **技能**：跨任务类型复用的子操作方法（如“系统性搜索容器并取物的方法”），从多个任务类型的轨迹中生成。
        *   **数据结构**：每个ECU是一个五元组 `(类型, 适用任务类型, 检索条件, 知识内容, 适应度分数)`。检索条件和知识内容分离，使检索更精准，注入给智能体的信息更纯粹。
    *   **适应度分数与生命周期管理**：
        *   采用拉普拉斯平滑分数 `fe = (s_e + 1) / (u_e + 2)` 追踪每个ECU的贡献，其中 `s_e` 是累计成功奖励，`u_e` 是使用次数。
        *   新创建的ECU初始分数为0.5，表示“未经证实”。
        *   完善的 **淘汰机制（Cleanup）** ：对于使用次数足够但分数低于阈值的ECU，以及同一（类型，任务类型）组内分数较低的冗余ECU，进行移除。
    *   **知识产出调度**：
        *   这是一个关键的资源分配模块，在每个演化周期动态决定当前应为哪类知识生成ECU。
        *   **三层调度逻辑**：
            1.  **可行性过滤**：跳过已存在足够高质量ECU的知识类型或任务类型，避免饱和。
            2.  **需求评估**：结合“任务类型覆盖缺口”和“平均知识质量缺口”来计算每种知识类型的当前需求。
            3.  **产出追踪**：使用指数移动平均追踪每种类型知识的历史生成“产出率”（即生成知识后对智能体表现提升的贡献），作为未来分配预算的参考。
        *   融合以上三层信号，为每种可行知识类型计算一个综合分数，并按此分数分配本轮固定的生成预算 `B`。
*   **核心算法流程（每个演化周期）**：
    1.  **评估**：使用当前ECU库，在独立的评估任务集上运行智能体，记录成功率，不更新任何分数。
    2.  **收集**：在与评估集不相交的收集任务集上运行智能体，记录完整轨迹与奖励，并根据这些轨迹更新ECU的适应度分数。
    3.  **调度**：执行KYS算法，根据当前库的状态、收集环节的成功率等信息，为下一阶段计算各知识类型的生成预算。
    4.  **生成**：根据分配到的预算，使用专门设计的提示词模板，调用大模型从收集的轨迹中提取相应类型的候选ECU，并通过一个“洗牌测试”过滤掉仅适用于特定实例的知识。
    5.  **清理**：执行淘汰机制，移除低质量和冗余ECU，形成下一周期的ECU库。

### 6. 主要结论与发现

*   **显著性能提升**：UCE将ALFWorld的成功率从 75.4% 提升至 **96.3%**，将WebShop的任务得分从 45.1% 提升至 **61.3%**，且WebShop的购买成功率从 30.0% 提升至 42.0%。
*   **知识互补是关键**：消融实验和案例研究表明，单一类型知识的增益有限，且在复杂任务（尤其是WebShop）中可能将智能体带入“死胡同”。不同类型的ECU协同作用，才能提供“正交”的信息，帮助智能体走出困境，这是性能大幅提升的核心原因。例如，WebShop的购物任务需要工作流、记忆（子页面只读）和策略（返回主页）三者结合才能成功。
*   **知识库可跨模型迁移**：UCE生成的ECU库并非依赖于特定的行动者格式，可以无损地迁移到 NoThinking、Plan-and-Act、ReflAct 等不同的行动者骨架上，并带来一致的性能提升。
*   **动态调整能力**：KYS能有效识别库中的知识缺口，并动态将生成预算分配给当前最需要的知识类型，使库的演化更加高效。

### 7. 优点

*   **创新性的知识类型分解**：将经验系统地、互补地分解为记忆、策略、工作流、技能四种类型，而非混合或单一处理，这在AI智能体持续学习领域是一个重要的概念性贡献。
*   **完整的生命周期管理**：为知识单元设计了完整的生成、评估、演化和淘汰闭环，特别是基于实际使用结果的**适应度分数**，比一次性评估更能反映知识的真实长期价值。
*   **智能的资源分配**：KYS调度机制是一个精巧的设计，融合了静态需求（覆盖缺口）和动态反馈（历史产出率），实现了生成预算的智能分配，具有“元学习”的色彩。
*   **无梯度与模型无关**：
    *   **无梯度**：方法简单、高效，可直接应用于任何闭源或开源LLM，无需昂贵的微调或RL训练。
    *   **模型格式无关**：生成的库可以跨多种推理格式的智能体迁移，展现了良好的通用性和鲁棒性。
*   **实验严谨且可解释性强**：实验设计有严格的隔离、公平的对比和统计检验。大量的案例研究和错误分析极大地增强了结论的可信度和可解释性。

### 8. 不足与局限

*   **知识类型的手工设计**：四种ECU类型是人工预定义的，这可能限制了其在新领域（如连续控制、多模态环境）的通用性。论文也指出未来可探索自动化类型发现。
*   **对环境的强依赖性**：
    *   实验仅集中在两个文本交互环境，这些环境具有离散动作空间、明确的文本状态和相对结构化的任务。其在更复杂、更开放或更长周期（如安卓设备操作AndroidWorld、视觉网页浏览VisualWebArena）的场景下的有效性有待考证。
    *   调度算法（KYS）的有效性目前仅在较小规模（100-134个任务）的任务池上得到验证，其在更大规模任务集上的可靠性和演化上限未知。
*   **生成质量的依赖性**：ECU的生成质量高度依赖于底层生成器模型（`gpt-5.2`）的推理和提取能力，框架的增益可能部分来自于强大生成器的功劳，和其他生成器模型搭配的效果需进一步评估。
*   **能力的边界**：错误分析揭示了该方法的一个能力边界。UCE擅长学习“是什么”（环境机制）和“做什么”（决策规则与流程），但对于需要精确执行“如何做”的任务（如从页面上的多个属性面板中精确选择一个特定尺寸或颜色），效果有限，这部分是梯度优化型强化学习的优势所在。

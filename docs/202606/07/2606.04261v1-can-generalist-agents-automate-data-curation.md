## 摘要
筛选训练数据是现代人工智能开发中最重要但也最费力的部分之一：从业者需要根据嘈杂的基准反馈迭代地提出、实施、评估和修改数据策略。我们探究通用编程智能体能否自动化这一数据筛选循环。我们引入了 *Curation-Bench*，一个以智能体为中心的基准测试，它固定了模型、训练配方和评估套件，同时赋予智能体命令行访问权限，以便检查数据、实施策略、将其提交到固定的训练/评估流水线并进行修改。在视觉-语言指令微调的实例化中，开箱即用的智能体在十次迭代内就达到了强大的已发表数据选择基线水平。然而，轨迹分析揭示了一个持续存在的 *执行-研究鸿沟*：智能体主要调整局部策略变体，而不是探索新的策略家族，即使给出了策略指南和论文参考文献也是如此。要求每次迭代引用、实例化并调整一种先前方法的支架将智能体转向方法引导的探索。被支架化的智能体在没有人类设计输入的情况下自主构建了一种数据选择策略，该策略在仅使用十分之一数据预算的情况下就超越了强大的已发表基线。总体而言，当前的智能体能够运行数据筛选循环，但可靠的数据研究需要支架化的方法调整，而不仅仅是开放式提示。代码和基准测试已开源。

## Abstract
Curating training data is among the most consequential yet labor-intensive parts of modern AI development: practitioners iteratively propose, implement, evaluate, and revise data policies against noisy benchmark feedback. We ask whether generalist coding agents can automate this data-curation loop. We introduce *Curation-Bench*, an agent-centric benchmark that fixes the model, training recipe, and evaluation suite while giving agents command-line access to inspect data, implement policies, submit them to a fixed training/evaluation pipeline, and revise. In a vision-language instruction-tuning instantiation, out-of-the-box agents reach strong published data-selection baselines within ten iterations. However, trajectory analysis reveals a persistent *execution-research gap*: agents mainly tune local policy variants rather than explore new policy families, even when given strategy guides and paper references. Scaffolds requiring each iteration to cite, instantiate, and adapt a prior method shift agents toward method-guided exploration. The scaffolded agent autonomously composes -- without human design input -- a data-selection policy that outperforms strong published baselines at one-tenth their data budget. Overall, current agents can run the curation loop, but reliable data research requires scaffolded method adaptation, not open-ended prompting alone. Code and benchmark are open-sourced.

*   **核心框架：Curation-Bench 基准**
    *   **设计原则**：
        1.  **数据隔离 (P1)**：固定模型架构、训练配方和评估套件。智能体唯一可操纵的变量是训练数据本身。
        2.  **终端真实性 (P2)**：智能体通过标准的命令行界面与工作区交互（检查文件、编写脚本、运行命令、调试）。
        3.  **污染控制 (P3)**：每个提交的数据集在训练前必须通过自动审计，防止评估数据泄露。
        4.  **轨迹可读性 (P4)**：完整记录每次迭代的策管脚本、数据清单、训练输出和评估日志。
*   **任务与流程**：一个任务 τ 被定义为 (初始模型 M₀, 候选数据池 Dpool, 数据预算 B, 固定训练流程 T, 评估套件 E, 约束 C)。
    *   智能体制定一个数据策略 π，该策略将池 Dpool 映射到一个不超过预算 B 的精炼数据集 Dπ。
    *   **智能体迭代循环**：在每次迭代 t，智能体执行“**检查数据 → 编写/修改策管脚本 → 生成数据集 → 提交验证与污染审查 → 触发固定训练和评估 → 接收分数与反馈 → 记录并计划下一步**”的标准循环。
*   **评估维度**：不仅评估最终得分，还首次引入**轨迹诊断**，对每次迭代进行四个维度的标注：

### **6. 论文的主要结论与发现**

*   **发现一：智能体是有效的执行者**。在开放式提示下，通用编程智能体能够可靠地运行数据筛选循环，并在 10 次迭代内达到或超越人类设计的强数据选择基线（如 ICONS, ARDS），以 1.5% 的数据恢复了约 60% 的全量数据微调增益。
*   **发现二：存在“执行-研究”鸿沟**。尽管结果良好，但轨迹诊断显示，开放式智能体的行为更像本地调优器而非研究者：它们主要调整源比例、长度阈值等本地启发式，而很少基于文献证据去探索全新的策略家族。
*   **发现三：轻型支架不足以改变行为**。提供策略列表或论文知识卡能拓宽智能体的“计划”，但**无法可靠地改变其“执行”策略**，它们仍倾向于选择实现成本最低的本地变体。
*   **发现四：重型支架是弥合鸿沟的关键**。通过强制要求每次迭代引用、实例化和调整先前的方法（方法适应支架），智能体被迫进行方法引导的探索。这种方式能有效消除浅层操作，提升新策略探索率，并最终使智能体自主组合出一种混合损失选择策略（EL2N 顶损失+助手损失噪声过滤），在**十分之一的数据预算**（10k）下超越了对比的 100k 数据基线的性能。
*   **发现五：策管搜索可扩展**。增加智能体的迭代次数（直至 50 次）能持续带来性能提升，表明在固定数据下，将计算投入“搜索更好数据子集”是一条有效的扩展路径。

### **7. 优点**

*   **新颖的基准与评估范式**：*Curation-Bench* 首次将数据策管表述为一个智能体策略搜索问题，并引入了**轨迹诊断**作为评估智能体“研究过程”质量的手段，而不仅看最终得分。
*   **深刻的洞见：“执行-研究鸿沟”**：精确地识别并命名了当前智能体在自动化科研中的核心瓶颈，即会做不会想，这为未来的智能体设计指明了方向。
*   **“支架”作为一种科学干预**：系统地研究了不同强度的提示/支架对智能体行为的影响，证明了通过设计任务协议可以塑造智能体的研究行为，这是一种重要的方法论贡献。
*   **实验的深度与广度**：在多模型、多数据集、多智能体主干、多支架级别和数据预算下进行了详尽的消融研究，实验结论扎实可靠。

### **8. 不足与局限**

*   **场景主要局限于微调**：虽然包含一个预训练实验，但核心实验主要围绕 VLM 的指令微调，其在纯文本 LLM 预训练或代码、数学等更专业领域的有效性有待验证。
*   **支架设计非正交**：重型支架混合了多种因素（方法接地、结构化日志、强制轮换），无法完全隔离出哪一个具体成分最关键，研究的是组合效应。
*   **轨迹标签引入主观性**：迭代的标签（如“shallow”）虽然有其严格标准，但本质上依赖于 LLM 或人类判断，可能引入评分者的偏见。
*   **评估噪声**：智能体在有噪声的基准反馈下工作，可能无意中过拟合了特定的评估套件，尽管存在污染控制，但这种风险依然存在。
*   **资源需求**：虽然降低了数据预算，但重型支架下的智能体搜索过程消耗了大量计算资源进行反复训练验证，对计算不充裕的团队是一种挑战。

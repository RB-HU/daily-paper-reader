## 摘要
电商搜索中的查询推荐旨在主动建议符合用户潜在兴趣的查询。然而，现有方法主要优化查询级别的相关性，而忽略了检索到的产品是否与用户的下游偏好一致。这种不匹配往往导致查询点击率（CTR）高但产品转化率（CVR）低。为弥合这一差异，我们提出QueryAgent-R1，一个记忆增强的智能体框架，通过链式检索优化来提升端到端对齐。我们的QueryAgent-R1将查询生成建立在真实库存检索之上，使智能体能够根据检索到的产品验证和优化查询。我们还在智能体强化学习（RL）过程中设计了一致性奖励，以联合优化查询相关性和下游参与度。此外，我们构建了一个记忆抽象模块以实现高效的用户画像。为支持离线评估，我们基于专有工业数据和公开数据集构建了两个数据集，QueryAgent-R1在这些数据集上持续优于强基线。此外，在大规模生产平台上，QueryAgent-R1在在线A/B测试中将查询CTR提高了2.9%，引导CVR提高了3.1%。

## Abstract
Query recommendation in e-commerce search aims to proactively suggest queries that match users' potential interests. However, existing methods mainly optimize query-level relevance, while neglecting whether the retrieved products align with users' downstream preferences. This mismatch often leads to high query click through rates (CTR) but low product conversion rates (CVR). To bridge this gap, we propose QueryAgent-R1, a memory-augmented agentic framework that improves end-to-end alignment via chain-of-retrieval optimization. Our QueryAgent-R1 grounds query generation in real inventory retrieval, allowing the agent to validate and refine queries based on retrieved products. We also design a consistency reward in the agentic reinforcement learning (RL) process to jointly optimize query relevance and downstream engagement. In addition, we construct a memory abstraction module for efficient user profiling. To support offline evaluation, we construct two datasets based on both proprietary industrial data and public datasets, on which QueryAgent-R1 consistently outperforms strong baselines. Moreover, on a large scale production platform, QueryAgent-R1 improves Query CTR by 2.9% and guided CVR by 3.1% in online A/B tests.

### 1. 论文的核心问题与整体含义 (研究动机和背景)

本论文关注电商搜索中一项关键但常被忽视的问题：**查询推荐**（Query Recommendation）仅在表面吸引用户点击，却未能有效引导用户完成深层次的购买转化。

*   **核心问题**：现有方法（如基于库存的协同过滤或基于语义的检索模型）主要优化**查询级别的相关性**，即推荐的查询词是否看起来有吸引力或与用户兴趣语义一致。这导致了一个“**生成-检索差距**”（Generation-Retrieval Gap）：推荐的查询可能获得高点击率（CTR），但检索出的产品与用户真实购买意图不符，导致**低产品转化率（CVR）**。
*   **研究目标**：弥合这个“点击”与“转化”之间的鸿沟，实现查询生成与下游产品偏好的**端到端对齐**。论文提出的“QueryAgent-R1”框架，旨在让系统在生成推荐查询时，不仅要考虑点击吸引力，更要预先验证该查询能否从真实商品库中检索出用户真正想购买的商品。

### 2. 论文提出的方法论 (核心思想、关键技术细节、算法流程)

论文的核心思想是构建一个**记忆增强的智能体框架**，该框架通过模拟“先检索、后验证、再优化”的链式过程，并结合精心设计的强化学习奖励，来联合优化查询点击和产品转化。

*   **记忆抽象机制 (Memory Abstraction Mechanism)**
    *   **目的**：高效地处理长达三个月、包含数万Token的原始用户行为日志，避免直接将海量文本注入模型上下文。
    *   **技术**：采用一个更大的模型（Qwen3-Next-80B-A3B）在离线状态下，将用户的原始行为异步压缩成一个紧凑、结构化的用户画像，包括最新意图、兴趣图谱、价格区间和潜在身份。当智能体工作时，通过调用 `Fetch_Memory` 工具直接读取这个缓存的画像，实现了约**10倍**的序列压缩。

*   **产品检索增强 (Product Retrieval Augmentation)**
    *   **目的**：将查询生成“接地”（Grounding），使模型在正式推荐前能在真实库存中验证查询的有效性。
    *   **技术**：构建一个包含400万个真实产品的“产品沙盒”。智能体在执行查询生成时，可调用 `Product_Search` 工具，该工具采用**混合检索策略**（BM25稀疏检索 + Qwen3-Embedding密集检索），并利用交叉编码器（Qwen3-Rerank）进行重排序。智能体据此检查生成查询所召回的产品，并依此对查询进行优化。

*   **智能体强化学习训练 (Agentic RL Training)**
    *   **目的**：通过强化学习，使策略模型能同时兼顾查询点击和产品命中，并以“一致性”为最终优化目标。
    *   **关键公式与流程**：
        *   **奖励函数 \( R(y) \)**：由格式奖励、工具调用奖励和命中奖励三部分组成。
        *   **一致性奖励 \( r_{\text{hit}}(y) \)**：进一步分解为查询奖励 \( r_q(y) \) 和产品奖励 \( r_i(y) \)。为了缓解奖励稀疏问题，每个子奖励都结合了**稀疏信号**（如查询精确匹配 \( \mathbb{I}(q_y = q^\star) \)）和**密集软信号**（如ROUGE相似度）。
        *   **优化算法**：采用**GDPO (Group Reward-Decoupled Normalization Policy Optimization)**。与GRPO相比，GDPO在不同的奖励成分上进行解耦归一化，能更好地处理本设计中存在多种异构信号（稀疏与密集）的情况，防止梯度干扰，提升训练稳定性。

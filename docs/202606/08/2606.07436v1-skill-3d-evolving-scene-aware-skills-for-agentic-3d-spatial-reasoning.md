## 摘要
本文探索了自主式3D空间理解，即多模态大语言模型智能体通过工具使用执行3D推理。现有方法经常误用工具，并在3D场景下表现出有偏的工具偏好，使得自主式范式相比非自主式策略仅带来微弱提升。我们发现3D空间推理任务在不同场景间具有异质性，而这些智能体对所有场景采用统一的工具使用策略，而非依据特定场景和任务选择工具。为解决此问题，我们提出Skill-3D，一个学习自演化场景感知技能的框架。具体而言，Skill-3D识别任务场景，将智能体的工具使用轨迹记录到场景记忆中，其中来自相似场景的成功轨迹被聚合并提炼为可复用的场景感知技能，失败的轨迹则作为教训附加于该技能。在训练过程中，当相似场景再次出现时，相应的技能被注入以引导智能体，产生新的轨迹，其成功与失败进一步优化该技能，形成一个记忆与技能库共同演化的循环。实验表明，Skill-3D显著提升了3D空间推理中的工具利用率（在VSI-Bench上从39%提高到78%），推动智能体实现正确且充分的工具使用。例如，它在MMSI-Bench上将Gemini-3-Flash的性能提升了67%。此外，我们基于技能引导的轨迹进行自主式后训练，将Qwen3-VL-8B在VSI-Bench上的表现提升了43%。

## Abstract
This paper explores agentic 3D spatial understanding, i.e., MLLM agents performing 3D reasoning through tool use. Existing methods often misuse tools and exhibit biased tool preferences under 3D scenarios, leaving the agentic paradigm with only marginal gains over non-agentic strategies. We reveal that 3D spatial reasoning tasks are heterogeneous across scenes, while these agents apply a uniform tool-use strategy to all scenes rather than selecting tools according to the specific scene and task. To address this, we propose Skill-3D, a framework that learns self-evolving scene-aware skills. Specifically, Skill-3D identifies the task scene and records the agent's tool-use trajectory into a Scene Memory, where successful trajectories from similar scenes are aggregated and distilled into a reusable scene-aware skill, with failed ones attached to the skill as lessons. During training, once a similar scene recurs, the corresponding skill is injected to guide the agent, producing new trajectories whose successes and failures further refine the skill, forming a loop in which the memory and the skill library co-evolve. Experiments show that Skill-3D substantially improves tool utilization in 3D spatial reasoning (from 39% to 78% on VSI-Bench), driving the agent toward correct and sufficient tool use. For instance, it improves Gemini-3-Flash by 67% on MMSI-Bench. Furthermore, we conduct agentic post-training over skill-guided trajectories, which boosts Qwen3-VL-8B by 43% on VSI-Bench.

### 2. 方法论

*   **核心思想**：
    *   构建一个**场景记忆库**，记录智能体在不同3D场景下的工具调用轨迹及其成败结果。在此基础上，演化出一个**技能库**，将相似场景下的成功经验提炼为可复用的“技能”，并将失败经验作为“教训”附加于相应技能之上。通过这种记忆与技能的**共同演化循环**，最终习得能够指导新任务的、场景感知的工具使用先验。

*   **关键技术细节与算法流程**：
    *   **框架名称**：**Skill-3D**
    *   **阶段一：场景感知技能提取 (Scene-Aware Skill Extraction)**
        1.  **轨迹记录与评估**：MLLM智能体执行一次完整的推理（即一次`rollout`），记录其推理链条、工具调用、工具输出和最终答案。根据答案正确与否，将该轨迹标记为“成功”或“失败”。
        2.  **成功轨迹 — 提炼为动态技能**：对于“成功”轨迹，提取其工具使用流程（触发条件、所需证据、工具顺序、关键参数等）。如果该流程与现有技能库不重复，则将其生成为一个新的**动态技能**；否则，用它来增强或合并到现有技能中。
        3.  **失败轨迹 — 转化为教训**：对于“失败”轨迹，分析其失败原因（如选错工具、忽略输出等），并将其作为**教训**附加到相关的技能上。当相似失败模式重复出现时，会为现有技能创建补丁规则，或生成专门处理该类失败的技能。
        4.  **技能维护**：由一个**技能管理器**负责维护技能库，确保其紧凑、可靠、无噪声，只接受有证据支持的更新。

    *   **阶段二：技能引导的推理 (Skill-Guided Inference)**
        1.  **场景-任务识别**：面对一个新问题时，智能体首先分析其场景特征和任务类型（如距离估计、物体计数等），确定所需证据。
        2.  **技能检索 (Skill Retrieval)**：根据当前场景-任务上下文，从技能库中检索出**Top-k**个最匹配的“静态技能”（任务级先验）和“动态技能”（场景适应流程）。匹配度基于语义、成功率、失败教训、工具成本等多维度评分。
        3.  **技能选择 (Skill Selection)**：由MLLM策略从候选中选出一个紧凑的技能子集，以确保覆盖所需证据的同时避免冗余工具调用。
        4.  **工具使用工作流**：在所选技能的指导下，智能体按步骤调用工具、整合证据，并最终将答案建立在获取的几何数据之上。

    *   **阶段三：技能引导的智能体后训练 (Skill-Guided Agentic Post-Training)**
        *   **目标**：将上述依赖庞大技能库的推理能力转移到紧凑的MLLM模型（如Qwen3-VL-8B）内部。
        *   **第一步：智能体监督微调**：使用技能引导的成功轨迹数据对目标模型进行SFT，使其学习技能检索、工具调用和证据整合的格式与行为。
        *   **第二步：智能体强化学习**：采用**GRPO**，使用一个复合奖励函数`R(τ) = Rans(τ) + Rfmt(τ) + Rtool(τ)`进行优化。其中，`Rtool(τ) = Rexec(τ) - |A|/B`，不仅奖励工具调用成功，还通过扣除项惩罚冗余的工具使用，以此鼓励模型内化高效、场景适配的工具使用策略。在此阶段，技能库被冻结。
